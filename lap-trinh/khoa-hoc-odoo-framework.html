CH 1 Odoo & kiến trúc

Hiểu bức tranh lớn trước khi viết dòng code đầu tiên.

Odoo vừa là một bộ ứng dụng quản trị doanh nghiệp (ERP) — Sales, Inventory, Accounting, HR, CRM... — vừa là một framework để bạn tùy biến và xây dựng ứng dụng riêng. Sức mạnh thật của Odoo nằm ở chỗ: mọi tính năng đều là module (addon) lắp ghép trên cùng một nền tảng, và bạn có thể kế thừa, mở rộng bất cứ phần nào mà không sửa code gốc.

Kiến trúc 3 tầng (MVC mở rộng)

TầngCông nghệVai trò
Dữ liệuPostgreSQLLưu trữ. Mỗi model ↔ một bảng.
Ứng dụng (server)Python + ORMModels, business logic, security — nơi bạn làm việc chính.
Giao diện (client)JavaScript (OWL), XML, QWebWeb client, views render trong trình duyệt.

Ánh xạ với MVC: Model = lớp Python (ORM) ↔ View = XML định nghĩa giao diện ↔ Controller = route HTTP. ORM của Odoo lo phần dịch giữa Python và SQL — bạn hiếm khi phải viết SQL tay.

Khái niệm cốt lõi cần nhớ

Thuật ngữLà gì
Module / AddonMột thư mục chứa code mở rộng Odoo (model, view, data...). Đơn vị cài/gỡ.
ModelLớp Python kế thừa models.Model, ánh xạ tới một bảng PostgreSQL.
RecordMột dòng dữ liệu (một bản ghi của model).
FieldMột cột/thuộc tính của model (Char, Many2one...).
RecordsetTập hợp các record cùng model — thứ bạn thao tác hằng ngày.
Environment (self.env)"Cổng" truy cập DB, model khác, user hiện tại, context.
ORMLớp ánh xạ đối tượng–quan hệ: thao tác DB bằng Python.

Community vs Enterprise

  • Community (LGPL, miễn phí): nhân framework + nhiều app cốt lõi. Đủ để học & xây dựng nhiều giải pháp.
  • Enterprise (thương mại): thêm app nâng cao (kế toán đầy đủ, studio, một số tính năng), giao diện bổ sung, hỗ trợ.
Triết lý "đừng sửa core"Quy tắc vàng của Odoo: không sửa trực tiếp module gốc. Thay vào đó, tạo module riêng và dùng cơ chế kế thừa (chương 13) để mở rộng. Nhờ vậy bạn nâng cấp Odoo mà không mất tùy biến.

CH 2 Cài đặt & môi trường dev

Dựng môi trường lập trình Odoo trên máy bạn.

Yêu cầu

  • Python (phiên bản theo bản Odoo — vd Odoo 17/18 cần Python 3.10+).
  • PostgreSQL (CSDL).
  • wkhtmltopdf (để xuất report PDF — cài bản đúng được Odoo khuyến nghị).

Cách cài cho dev: chạy từ mã nguồn

terminal
# 1) Lấy mã nguồn (chọn đúng nhánh phiên bản, vd 17.0)
git clone https://github.com/odoo/odoo --depth 1 --branch 17.0 odoo

# 2) Tạo môi trường ảo Python & cài thư viện
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r odoo/requirements.txt

# 3) Tạo user PostgreSQL cho Odoo
createuser -s odoo              # hoặc tạo qua psql

# 4) Chạy Odoo: tạo DB tên 'dev' và cài sẵn module base
python odoo/odoo-bin -d dev -i base --addons-path=odoo/addons,custom_addons

Mở trình duyệt tại http://localhost:8069. Lựa chọn khác: dùng Docker (image odoo + postgres) rất tiện để cô lập môi trường.

File cấu hình odoo.conf

odoo.conf

Chạy với cấu hình: python odoo/odoo-bin -c odoo.conf.

Các cờ dòng lệnh hay dùng

CờÝ nghĩa
-d <db>Chọn/tạo database
-i <module>Cài (install) module
-u <module>Nâng cấp (upgrade) module — chạy mỗi khi bạn sửa Python/XML

Tạo nhanh khung moduleOdoo có sẵn lệnh scaffold: python odoo-bin scaffold my_module custom_addons/ tạo cấu trúc mẫu. Hoặc dùng 🧩 Scaffold Generator của khóa học (góc dưới phải) để sinh khung kèm fields tùy ý.
Nhớ nâng cấp moduleSửa code Python/XML xong, Odoo không tự nhận mọi thay đổi (đặc biệt là dữ liệu XML). Hãy nâng cấp: -u my_module (hoặc bấm Upgrade trong giao diện Apps). Với --dev=all, thay đổi Python & view thường được reload, nhưng đổi cấu trúc field/data thì vẫn cần -u.

CH 3 Cấu trúc một module

Giải phẫu một addon: thư mục, manifest và vòng đời.

Cây thư mục chuẩn

my_module/
my_module/
├── __init__.py            # import các package con (models, ...)
├── __manifest__.py        # "khai sinh" module: tên, depends, data...
├── models/                # mã Python: models & logic
│   ├── __init__.py
│   └── library_book.py
├── views/                 # giao diện XML: form/list/menu/action
│   └── library_book_views.xml
├── security/              # phân quyền
│   ├── ir.model.access.csv
│   └── security.xml        # groups & record rules (tùy)
├── data/                  # dữ liệu khởi tạo (XML/CSV)
├── demo/                  # dữ liệu demo
├── wizard/                # TransientModel (hộp thoại)
├── report/                # mẫu report QWeb
├── controllers/           # route HTTP / API
├── static/                # JS, CSS, hình ảnh (frontend)
└── i18n/                  # file dịch .po

__manifest__.py — trái tim của module

__manifest__.py
{
    "name": "My Library",            # tên hiển thị
    "version": "1.0",
    "summary": "Quản lý thư viện sách",
    "author": "Your Name",
    "category": "Services",
    "license": "LGPL-3",
    "depends": ["base"],       # module phụ thuộc (LUÔN cần)
    "data": [                      # file XML/CSV nạp khi cài (THỨ TỰ quan trọng)
        "security/ir.model.access.csv",
        "views/library_book_views.xml",
    ],
    "demo": ["demo/demo.xml"],  # chỉ nạp khi bật demo data
    "application": True,         # hiện như một "App" độc lập
    "installable": True,
}
Thứ tự trong "data" rất quan trọngFile được nạp theo thứ tự liệt kê. security phải đứng trước view dùng tới model đó; view tham chiếu action thì action phải nạp trước. Sai thứ tự → lỗi "External ID not found" khi cài.

Vòng đời module

  • Install (-i): tạo bảng từ models, nạp file trong data, tạo external ID.
  • Upgrade (-u): áp thay đổi cấu trúc & nạp lại data (data không noupdate).
  • Uninstall: gỡ bảng & dữ liệu liên quan.
  • depends: Odoo tự cài module phụ thuộc trước; thứ tự nạp tuân theo cây phụ thuộc.

from odoo import models, fields


class LibraryBook(models.Model):

CH 5 Models

Lớp Python ánh xạ tới bảng dữ liệu — viên gạch nền của mọi thứ.

models/library_book.py
from odoo import models, fields


class LibraryBook(models.Model):
    _name = "library.book"
    _description = "Library Book"      # bắt buộc (nếu không sẽ cảnh báo)
    _order = "name asc"               # thứ tự mặc định khi truy vấn
    _rec_name = "name"                 # field dùng làm "tên hiển thị" (mặc định là 'name')

    name = fields.Char(required=True)

Các thuộc tính _ quan trọng

Thuộc tínhÝ nghĩa
_nameTên kỹ thuật model (dấu chấm). library.book → bảng library_book.
_descriptionMô tả người đọc hiểu. Nên luôn có.
_orderSắp xếp mặc định (vd "create_date desc").
_rec_nameField hiển thị thay cho tên record (mặc định name).
_inheritKế thừa/mở rộng model khác (chương 13).
_sql_constraintsRàng buộc ở tầng DB (chương 12).

Các loại model

Lớp cơ sởDùng cho

CH 7 Relational fields (quan hệ)

Liên kết các model — nơi mô hình dữ liệu thật sự thành hình.

FieldQuan hệVí dụ
Many2oneNhiều → một (khóa ngoại)Mỗi sách thuộc một tác giả
One2manyMột → nhiều (ảo, ngược của Many2one)Một tác giả có nhiều sách
Many2manyNhiều ↔ nhiều (bảng trung gian)Sách có nhiều thể loại & ngược lại
relations.py
class LibraryBook(models.Model):
    _name = "library.book"

    name = fields.Char(required=True)
    # Many2one: lưu khóa ngoại tới res.partner
    author_id = fields.Many2one("res.partner", string="Tác giả",
                                ondelete="set null", index=True)
    # Many2many: chọn nhiều thể loại
    category_ids = fields.Many2many("library.category", string="Thể loại")


class LibraryAuthorMixinExample(models.Model):
    _name = "library.author"

    name = fields.Char(required=True)
    # One2many: ảo, cần trỏ ngược về field Many2one ở library.book
    book_ids = fields.One2many("library.book", "author_id", string="Sách")
One2many là "ảo"One2many không tạo cột trong DB — nó chỉ là "view ngược" của một Many2one bên model kia. Vì vậy nó cần 2 tham số: model đích và tên field Many2one trỏ ngược lại ("author_id"). Many2many tạo một bảng trung gian tự động.

ondelete cho Many2one

  • "set null" (mặc định): xóa bản ghi cha → field thành rỗng.
  • "restrict": chặn xóa nếu còn bản ghi con tham chiếu.
  • "cascade": xóa cha → xóa luôn con. Dùng thận trọng.

Command — gán giá trị cho One2many / Many2many

Vì đây là tập record, bạn dùng "Command" để thêm/bớt/liên kết:

commands.py
from odoo import Command

book.write({
    "category_ids": [
        Command.set([cat1.id, cat2.id]),   # thay toàn bộ bằng tập này
        # Command.link(id)    → liên kết thêm 1 record có sẵn
        # Command.unlink(id)  → bỏ liên kết (không xóa record)
        # Command.create({...}) → tạo mới record con & liên kết
        # Command.clear()     → bỏ hết liên kết
    ]
})
Khác biệt phiên bảnCú pháp cũ dùng tuple số: (6, 0, [ids]) = set, (4, id) = link, (0, 0, {vals}) = create... Bản Odoo mới khuyến nghị dùng Command.* cho dễ đọc — cả hai vẫn chạy.

CH 8 Computed, related & onchange

Field "thông minh": tự tính, tự lấy từ quan hệ, hoặc gợi ý khi nhập liệu.

Computed field

computed.py
from odoo import models, fields, api


class SaleLine(models.Model):
    _name = "library.loan"

    qty = fields.Integer(default=1)
    unit_price = fields.Float()
    total = fields.Float(string="Thành tiền", compute="_compute_total", store=True)

    @api.depends("qty", "unit_price")   # tính lại khi 1 trong các field này đổi
    def _compute_total(self):
        for rec in self:           # LUÔN lặp qua self (recordset)
            rec.total = rec.qty * rec.unit_price
3 quy tắc của computed field
  1. Luôn lặp for rec in self và gán cho từng rec (self có thể là nhiều record).
  2. @api.depends(...) liệt kê mọi field mà phép tính phụ thuộc (kể cả qua quan hệ: "line_ids.total").
  3. store=True để lưu vào DB (tìm/sắp xếp được, tính lại khi depends đổi). Không store → tính mỗi lần đọc, không search được (trừ khi viết hàm search).

Mọi thao tác trả về một recordset — tập 0..n record cùng model. self trong method chính là một recordset. Nó hỗ trợ lặp, lọc, gộp như collection.

CH 11 Methods & decorators

Viết logic nghiệp vụ — và hiểu các decorator @api quyết định cách method chạy.

methods.py
from odoo import models, fields, api
from odoo.exceptions import UserError


class LibraryBook(models.Model):
    _name = "library.book"

    name = fields.Char(required=True)
    state = fields.Selection([("draft","Nháp"),("available","Có sẵn"),
                              ("borrowed","Đang mượn")], default="draft")

    # method thường: self là recordset → lặp qua từng record
    def action_borrow(self):
        for book in self:
            if book.state != "available":
                raise UserError(f"Sách '{book.name}' không sẵn để mượn.")
            book.state = "borrowed"
        return True

Method "action" như trên thường được gắn vào nút bấm trên form (chương 16): <button name="action_borrow" type="object"/>.

Ghi đè create / write (super)

override.py
    @api.model_create_multi
    def create(self, vals_list):
        for vals in vals_list:
            if not vals.get("name"):
                vals["name"] = "Sách mới"
        records = super().create(vals_list)      # gọi logic gốc
        # ... việc sau khi tạo ...
        return records

    def write(self, vals):
        res = super().write(vals)
        # ... việc sau khi ghi ...
        return res
Luôn gọi super()Khi ghi đè create/write/unlink, phải gọi super() để giữ hành vi gốc (và để các module khác cũng ghi đè được). Quên super() = vỡ chuỗi kế thừa.
Khác biệt phiên bảnBản mới khuyến nghị @api.model_create_multi với create(self, vals_list) (nhận list). Bản cũ dùng @api.model + create(self, vals) (một dict). Viết theo kiểu list giúp tạo hàng loạt hiệu quả.

from odoo.exceptions import ValidationError

    @api.constrains("price", "pages")
    def _check_values(self):
        for rec in self:
            if rec.price < 0:
                raise ValidationError("Giá không được âm.")
            if rec.pages and rec.pages > 10000:
                raise ValidationError(f"Số trang ({rec.pages}) không hợp lý.")

class ResPartner(models.Model):

# Lấy record theo External ID
cat = self.env.ref("my_library.category_tech")

# Sinh mã tự động (số phiếu, mã đơn...) bằng ir.sequence
    code = fields.Char(default=lambda self: self.env["ir.sequence"]
                       .next_by_code("library.book.code") or "NEW")

Định nghĩa sequence trong data XML (model ir.sequence) với code, prefix, padding. CSV cũng dùng để nạp dữ liệu hàng loạt (đặt file .csv tên theo model trong data).

CH 19 Wizards (TransientModel)

Hộp thoại tương tác nhiều bước — nhập liệu tạm rồi xử lý.

Wizard là model kế thừa models.TransientModel: có form như model thường nhưng dữ liệu tạm thời (tự dọn). Dùng cho thao tác hàng loạt, nhập tham số, xác nhận.

wizard/bulk_price.py
class BulkPriceWizard(models.TransientModel):
    _name = "library.bulk.price.wizard"
    _description = "Cập nhật giá hàng loạt"

    percent = fields.Float(string="% thay đổi", required=True)

    def action_apply(self):
        self.ensure_one()
        # lấy các record được chọn từ context (active_ids)
        books = self.env["library.book"].browse(self.env.context.get("active_ids", []))
        for book in books:
            book.price = book.price * (1 + self.percent / 100)
        return {"type": "ir.actions.act_window_close"}
active_id / active_ids / active_modelKhi mở wizard từ một danh sách/record, Odoo truyền các record đang chọn vào context: active_ids (danh sách id), active_id (id hiện tại), active_model. Đây là cách wizard "biết" nó đang xử lý gì.

View của wizard là một <form> với <footer> chứa nút gọi action_apply; mở bằng action act_window có target="new" (popup).

mail.thread & chatter (tracking + tin nhắn)

class LibraryBook(models.Model):
    _name = "library.book"
    _inherit = ["mail.thread", "mail.activity.mixin"]   # thêm chatter + activity

    name = fields.Char(required=True, tracking=True)   # tracking: log khi đổi
    state = fields.Selection([...], tracking=True)

    def action_notify(self):
        self.message_post(body="Sách đã được cập nhật.")      # đăng tin lên chatter

Thêm vào form view: <chatter/> (bản mới) hoặc khối <div class="oe_chatter"> với message_ids/activity_ids/message_follower_ids (bản cũ) để hiện hộp trao đổi, lịch sử thay đổi và hoạt động (activity).

Controller — route web/HTTP

from odoo import http
from odoo.http import request


class LibraryController(http.Controller):

    @http.route("/library/books", type="http", auth="public", website=True)
    def list_books(self, **kw):
        books = request.env["library.book"].sudo().search([])
        return request.render("my_library.books_page", {"books": books})

    @http.route("/api/books", type="json", auth="user")
    def api_books(self, **kw):
        books = request.env["library.book"].search_read([], ["name", "price"])
        return {"books": books}

External API — gọi Odoo từ ngoài (XML-RPC/JSON-RPC)

import xmlrpc.client

url, db, user, pwd = "http://localhost:8069", "dev", "admin", "admin"
common = xmlrpc.client.ServerProxy(f"{url}/xmlrpc/2/common")
uid = common.authenticate(db, user, pwd, {})

models = xmlrpc.client.ServerProxy(f"{url}/xmlrpc/2/object")
# gọi search_read trên library.book
books = models.execute_kw(db, uid, pwd,
    "library.book", "search_read",
    [[("price", ">", 20)]], {"fields": ["name", "price"], "limit": 10})
print(books)
API = chính ORMExternal API gọi đúng các method ORM bạn đã học (search_read, create, write, unlink, hay bất kỳ method public nào của model). Quyền vẫn áp theo user đăng nhập. Đây là cách tích hợp Odoo với web/app/hệ thống khác.

CH 24 Deploy & migration

Đưa module lên production và sống sót qua các lần nâng cấp.

Quản lý phiên bản module

  • version trong manifest theo quy ước <odoo>.<major>.<minor>.<patch> (vd 17.0.1.0.0).
  • Đặt code trong git; mỗi nhánh tương ứng một phiên bản Odoo.
  • Tách custom addons khỏi source Odoo (thư mục riêng trong addons_path).

Migration scripts (đổi cấu trúc dữ liệu)

Khi nâng version module mà cần biến đổi dữ liệu cũ, đặt script vào thư mục migrations/<version>/:

migrations/17.0.2.0.0/pre-migrate.py
def migrate(cr, version):
    # cr = con trỏ DB; chạy SQL để chuyển đổi dữ liệu cũ
    cr.execute("UPDATE library_book SET state = 'available' WHERE state IS NULL")

pre-migrate.py chạy trước khi cập nhật schema, post-migrate.py chạy sau. Công cụ OpenUpgrade (OCA) hỗ trợ nâng cấp giữa các phiên bản Odoo lớn.
