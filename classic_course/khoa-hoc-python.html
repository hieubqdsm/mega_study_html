CH 1 Python & thiết lập môi trường

Python là gì, vì sao học, cài đặt, chạy dòng code đầu tiên.

Python là gì & tại sao học?

Python là ngôn ngữ lập trình bậc cao, thông dịch (interpreted), gõ động (dynamic typing), nổi tiếng vì cú pháp gần với tiếng Anh và dễ đọc. Nó là ngôn ngữ số 1 cho Data/AI, rất mạnh cho backend web (Django, FastAPI), automation/scripting, DevOps, và là ngôn ngữ tuyệt vời để bắt đầu lập trình.

  • Interpreted: chạy trực tiếp từng dòng, không cần biên dịch (compile) như C/Java.
  • Dynamic typing: không khai báo kiểu trước; x = 5 rồi x = "hi" đều hợp lệ.
  • "Batteries included": thư viện chuẩn cực giàu (file, web, toán, ngày giờ...).

Cài đặt

  1. Tải Python 3.12+ tại python.org/downloads. Windows: nhớ tick "Add Python to PATH".
  2. Cài VS Code + extension "Python" của Microsoft.
  3. Kiểm tra trong terminal: python --version (Windows) hoặc python3 --version (macOS/Linux).
Chạy code3 cách: (1) file .py → python ten_file.py; (2) REPL — gõ python để vào chế độ tương tác; (3) ngay trong khóa học này bằng nút ▶ Run.

CH 8 Dict & Set

Dict: ánh xạ key→value (cực kỳ quan trọng). Set: tập hợp không trùng.

Dictionary

dict.py
sv = {"ten": "An", "tuoi": 20, "diem": [8, 9]}
print(sv["ten"])             # An
print(sv.get("sdt", "N/A"))   # an toàn nếu key thiếu
sv["email"] = "an@x.com"    # thêm/cập nhật

for key, val in sv.items():
    print(key, "=>", val)

# Đếm tần suất - pattern siêu phổ biến
text = "abracadabra"
dem = {}
for ch in text:
    dem[ch] = dem.get(ch, 0) + 1
print(dem)

Set — tập hợp

set.py
a = {1, 2, 3, 3}     # trùng tự loại -> {1,2,3}
b = {2, 3, 4}
print(a | b)            # hợp {1,2,3,4}
print(a & b)            # giao {2,3}
print(a - b)            # hiệu {1}
# Loại trùng trong list cực nhanh:
print(list(set([1,1,2,3,3])))
Khi nào dùng gì?List: dãy có thứ tự, cho phép trùng. Dict: tra cứu theo key (O(1)). Set: kiểm tra "đã tồn tại chưa" & loại trùng (O(1)). Tuple: dữ liệu cố định, làm key của dict.

CH 9 Hàm (Function)

Đóng gói logic tái sử dụng — viên gạch nền của mọi chương trình.

func.py
def chao(ten, loi="Xin chào"):      # loi: tham số mặc định
    return f"{loi}, {ten}!"

print(chao("An"))
print(chao("Bình", loi="Hi"))   # keyword argument

# *args (nhiều vị trí), **kwargs (nhiều keyword)
def tong(*nums):
    return sum(nums)
print(tong(1, 2, 3, 4))   # 10

def info(**kw):
    return kw
print(info(ten="An", tuoi=20))
Bẫy kinh điển: mutable defaultĐừng dùng list/dict làm giá trị mặc định: def f(x, acc=[]): — list này được chia sẻ giữa các lần gọi. Dùng def f(x, acc=None): acc = acc or [].
DocstringDòng chuỗi đầu hàm mô tả công dụng: """Trả về tổng...""". Truy cập qua help(ham) hoặc ham.__doc__. Tập thói quen viết từ sớm.

def do_thoi_gian(func):
    @functools.wraps(func)          # giữ tên/docstring gốc
    def wrapper(*args, **kwargs):
        t0 = time.perf_counter()
        kq = func(*args, **kwargs)

import functools

def retry(n):
    def deco(func):
        @functools.wraps(func)
        def wrapper(*a, **k):
            for lan in range(1, n + 1):
                try:
                    return func(*a, **k)
                except Exception as e:
                    print(f"  lần {lan} lỗi: {e}")
                    if lan == n: raise
        return wrapper
    return deco

import asyncio

async def tai(ten, giay):
    await asyncio.sleep(giay)        # giả lập I/O không chặn

from abc import ABC, abstractmethod

# --- Strategy: hoán đổi thuật toán lúc chạy ---
class GiamGia(ABC):
    @abstractmethod
    def tinh(self, gia): ...

class KhongGiam(GiamGia):
    def tinh(self, gia): return gia
class GiamPhanTram(GiamGia):
    def __init__(self, pct): self.pct = pct

    def __init__(self, gia, cs: GiamGia):
        self.gia, self.cs = gia, cs

CH 30 Async nâng cao

async_advanced.py
import asyncio

async def worker(name, q, sem):
    while not q.empty():
        item = await q.get()
        async with sem:                  # giới hạn đồng thời
            await asyncio.sleep(0.1)         # giả lập gọi API
            print(f"{name} xử lý {item}")
        q.task_done()

async def main():
    q = asyncio.Queue()
    for i in range(6): q.put_nowait(i)
    sem = asyncio.Semaphore(2)            # tối đa 2 cùng lúc
    workers = [asyncio.create_task(worker(f"W{n}", q, sem)) for n in range(3)]
    await q.join()
    for w in workers: w.cancel()

asyncio.run(main())
Bẫy async phổ biến(1) Gọi hàm blocking (vd requests, time.sleep) trong coroutine → chặn cả event loop. Dùng thư viện async (httpx, asyncio.sleep) hoặc run_in_executor. (2) Quên await → coroutine không chạy. (3) Không xử lý CancelledError/timeout.
Góc seniorasyncio.Semaphore để rate-limit; asyncio.timeout() (3.11+) cho deadline; TaskGroup (3.11+) để quản lý nhóm task an toàn (tự hủy khi 1 cái lỗi). Hiểu rõ structured concurrency.

import re
text = "Đơn #1024 ngày 2024-03-15, đơn #2048 ngày 2024-11-02"

# findall: lấy tất cả khớp
print(re.findall(r"#(\d+)", text))         # ['1024', '2048'] (group 1)

m = re.search(r"(\d{4})-(\d{2})-(\d{2})", text)

print(re.sub(r"#\d+", "#****", text))

m = re.search(r"(?P<y>\d{4})-(?P<m>\d{2})-(?P<d>\d{2})", text)
print(m.group("y"), m.groupdict())

import re
# compile 1 lần, dùng nhiều lần (nhanh hơn trong vòng lặp lớn)
PAT = re.compile(r"<(\w+)>")

html = "<div><span>hi</span></div>"

for m in PAT.finditer(html):
    print(m.group(1), "tại", m.start())

    return p

    seen = {}
    for i, x in enumerate(nums):

        seen[x] = i

B8 · LRU Cache LeetCode Medium · hỏi cực nhiều ở Mid

Cài LRUCache(capacity) với get(key) & put(key, value) đều O(1). Khi đầy, loại phần tử ít dùng gần đây nhất.

Gợi ý
collections.OrderedDict: move_to_end khi truy cập, popitem(last=False) khi đầy. (Hoặc tự cài double-linked list + dict nếu họ cấm dùng thư viện.)
Đáp án
sol.py
from collections import OrderedDict
class LRUCache:
    def __init__(self, cap):
        self.cap = cap; self.d = OrderedDict()
    def get(self, key):
        if key not in self.d: return -1
        self.d.move_to_end(key)
        return self.d[key]
    def put(self, key, val):
        if key in self.d: self.d.move_to_end(key)
        self.d[key] = val
        if len(self.d) > self.cap:
            self.d.popitem(last=False)   # bỏ cái cũ nhất
c = LRUCache(2)
c.put(1,1); c.put(2,2); print(c.get(1))
c.put(3,3); print(c.get(2))   # -1 (đã bị loại)
B9 · Rate limiter phỏng vấn backend Mid/Senior

Cài RateLimiter(max_calls, per_seconds): trả True nếu được phép gọi, False nếu vượt hạn mức trong cửa sổ thời gian. (Sliding window.)

Đáp án
sol.py
from collections import deque
class RateLimiter:
    def __init__(self, max_calls, per):
        self.max, self.per = max_calls, per
        self.calls = deque()
    def allow(self, now):
        while self.calls and now - self.calls[0] >= self.per:
            self.calls.popleft()          # bỏ call ngoài cửa sổ
        if len(self.calls) < self.max:
            self.calls.append(now); return True
        return False
rl = RateLimiter(2, 10)
print([rl.allow(t) for t in [0, 1, 2, 11]])   # [True, True, False, True]

import functools, time
def timed_cache(seconds):
    def deco(func):
        store = {}    # key -> (value, expire_at)
        @functools.wraps(func)
        def wrapper(*args):
            now = time.monotonic()
            if args in store and store[args][1] > now:
                return store[args][0]
            val = func(*args)
            store[args] = (val, now + seconds)
            return val
        return wrapper
    return deco

@timed_cache(60)

  2. Khi nào dùng asyncio vs multiprocessing vs ThreadPoolExecutor? Cho ví dụ mỗi loại.
  3. Mô tả cách bạn thiết lập CI/CD cho team Python (các bước bắt buộc trước merge).
  4. Một junior viết hàm 200 dòng làm 5 việc. Bạn review thế nào? (nói về SOLID, cách góp ý)
Đáp án Phần B + tiêu chí Senior

Phần B — các vấn đề:

  1. (1) SQL Injection — nối f-string vào SQL. Sửa: tham số hóa "... WHERE id = ?", (id,). Đồng thời là vấn đề N+1 query: nên một câu WHERE id IN (...) thay vì truy vấn trong vòng lặp.
  2. (2) Không xử lý khi id không tồn tại; nên dùng generator/streaming nếu ids lớn; id trùng tên built-in (đặt user_id).
  3. (3) Mutable default data=[] — chia sẻ giữa các lần gọi. Dùng data=None.
  4. (4) Bare except + pass — nuốt mọi lỗi (kể cả KeyboardInterrupt), che giấu bug. Bắt đúng loại, log lại, xử lý có ý nghĩa.

Đạt Early Senior nếu: Phần A có cấu trúc (queue + worker + idempotency + retry + observability) và biết nói về đánh đổi & SPOF; Phần B tìm ≥4 lỗi và phân loại được mức độ (bảo mật > đúng đắn > phong cách); Phần C thể hiện tư duy "đo trước khi tối ưu", chọn mô hình concurrency đúng theo bản chất tác vụ, và cách mentor/giao tiếp khi review — không chỉ kỹ thuật.

Sau khi làm hết 3 đềNếu bạn tự tin với đề tương ứng cấp độ → bạn đã sẵn sàng phỏng vấn vị trí đó. Ghi âm lại phần "nói" (design/review) và nghe lại — phỏng vấn Senior chấm cách bạn lập luận nhiều như kết quả cuối.

🃏 Flashcards ôn nhanh trước phỏng vấn

Câu hỏi bật nhanh hay gặp ở vòng hỏi miệng. Bấm vào thẻ để lật xem đáp án. Lọc theo cấp độ để ôn đúng trọng tâm.

28 thẻ
NEWBIE
Phân biệt list và tuple?
ĐÁP ÁN
list mutable (sửa được), dùng cho dãy thay đổi. tuple immutable (cố định), nhẹ hơn, làm được key của dict. Dùng tuple cho dữ liệu không đổi (tọa độ, record).
NEWBIE
is khác == thế nào?
ĐÁP ÁN
== so sánh giá trị; is so sánh danh tính (cùng object trong bộ nhớ). Dùng is chỉ với None/True/False.
NEWBIE
Những giá trị nào là "falsy"?
ĐÁP ÁN
0, 0.0, "", [], {}, (), set(), None, False. Nên viết if my_list: thay vì if len(my_list) > 0:.
NEWBIE
"5" + 3 ra gì?
ĐÁP ÁN
TypeError — không cộng str với int. Phải ép kiểu int("5") + 3. Dữ liệu từ input() luôn là str.
NEWBIE
7 / 2, 7 // 2, 7 % 2 ra gì?
ĐÁP ÁN
3.5 (chia luôn float), 3 (chia lấy nguyên), 1 (chia lấy dư).
NEWBIE
*args và **kwargs là gì?
ĐÁP ÁN
*args gom nhiều tham số vị trí thành tuple; **kwargs gom nhiều tham số keyword thành dict. Cho hàm nhận số lượng đối số linh hoạt.
NEWBIE
Khi nào dùng set?
ĐÁP ÁN
Loại trùng (list(set(xs))) và kiểm tra "đã tồn tại chưa" với độ phức tạp O(1). Phép hợp/giao/hiệu: | & -.
JUNIOR
Shallow copy vs deep copy?
ĐÁP ÁN
a.copy()/a[:] copy nông — object lồng bên trong vẫn dùng chung. copy.deepcopy() copy sâu toàn bộ. b = a không copy gì (cùng tham chiếu).
JUNIOR
Bẫy "mutable default argument" là gì?
ĐÁP ÁN
def f(x, acc=[]): list mặc định tạo 1 lần, chia sẻ giữa các lần gọi → tích lũy ngoài ý muốn. Sửa: acc=None; acc = acc or [].
JUNIOR
Vì sao tra cứu trong set/dict nhanh hơn list?
ĐÁP ÁN
x in list là O(n) (quét tuần tự). x in set/dict là O(1) trung bình nhờ bảng băm (hash table).
JUNIOR
@property dùng để làm gì?
ĐÁP ÁN
Cho phép truy cập một method như thuộc tính (không cần ()), kèm logic getter/setter (validate, tính toán) mà không đổi giao diện gọi.
JUNIOR
Thứ tự chạy try / except / else / finally?
ĐÁP ÁN
try → nếu lỗi vào except, nếu không vào else → finally luôn chạy cuối cùng (dù lỗi hay không, kể cả khi return).
JUNIOR
List comprehension vs generator expression?
ĐÁP ÁN
[...] tạo cả list trong RAM ngay. (...) tạo generator lười, sinh từng phần tử — tiết kiệm bộ nhớ với dữ liệu lớn.
JUNIOR
EAFP là gì?
ĐÁP ÁN
"Easier to Ask Forgiveness than Permission" — cứ thử rồi bắt lỗi (try/except), thay vì kiểm tra mọi điều kiện trước (LBYL). Phong cách Pythonic.
MID
GIL là gì? Ảnh hưởng ra sao?
ĐÁP ÁN
Global Interpreter Lock — CPython chỉ cho 1 thread chạy bytecode tại một thời điểm. → threading vô dụng cho CPU-bound (dùng multiprocessing), nhưng tốt cho I/O-bound.
MID
threading vs multiprocessing vs asyncio — chọn khi nào?
ĐÁP ÁN
asyncio: I/O-bound số lượng lớn (nhiều request mạng). multiprocessing: CPU-bound (tính toán nặng). threading/ThreadPool: I/O-bound đơn giản, ít task.
MID
Decorator hoạt động thế nào? Vì sao cần functools.wraps?
ĐÁP ÁN
Decorator nhận một hàm, trả về hàm bọc (thêm hành vi). @wraps giữ __name__/__doc__ của hàm gốc để debug & introspection không bị mất.
MID
yield làm gì?
ĐÁP ÁN
Biến hàm thành generator: trả tạm 1 giá trị & tạm dừng, lần gọi sau tiếp tục từ đó. Sinh dữ liệu lười, tiết kiệm RAM, xử lý luồng vô hạn.
MID
@staticmethod vs @classmethod vs method thường?
ĐÁP ÁN
Thường nhận self (instance). classmethod nhận cls (factory, thao tác trên class). staticmethod không nhận gì (hàm tiện ích gắn vào class theo ngữ nghĩa).
MID
__str__ vs __repr__?
ĐÁP ÁN
__str__ cho người dùng cuối (dễ đọc). __repr__ cho lập trình viên (rõ ràng, lý tưởng tái tạo được object — dùng khi debug/log/REPL).
MID
Closure là gì?
ĐÁP ÁN
Hàm con "nhớ" được biến từ phạm vi bao quanh nó ngay cả khi hàm ngoài đã kết thúc. Nền tảng của decorator & factory functions.
MID
Type hints có được kiểm tra lúc chạy không?
ĐÁP ÁN
Không. Python bỏ qua khi chạy (chỉ là chú thích). Kiểm tra bằng công cụ tĩnh mypy/pyright. Pydantic thì validate runtime (khác cơ chế).
MID
Context manager / with để làm gì?
ĐÁP ÁN
Đảm bảo dọn dẹp tài nguyên (đóng file, trả connection, release lock) kể cả khi có lỗi, qua __enter__/__exit__ hoặc @contextmanager.
SENIOR
SOLID gồm những gì?
ĐÁP ÁN
Single Responsibility · Open/Closed (mở mở rộng, đóng sửa) · Liskov (lớp con thay được cha) · Interface Segregation · Dependency Inversion (phụ thuộc abstraction).
SENIOR
Vấn đề N+1 query là gì?
ĐÁP ÁN
Truy vấn 1 lần lấy N bản ghi, rồi lặp N lần mỗi cái 1 query phụ → 1+N truy vấn. Sửa: dùng JOIN/WHERE IN (...) hoặc eager loading (selectinload).
SENIOR
Quy trình tối ưu một service chậm?
ĐÁP ÁN
(1) Đặt mục tiêu đo được → (2) profile tìm nút thắt thật (80/20) → (3) tối ưu thuật toán/cấu trúc dữ liệu → (4) cache → (5) cuối cùng mới song song/C-ext. Đo, đừng đoán.
SENIOR
__slots__ dùng để làm gì?
ĐÁP ÁN
Khai báo cố định tập thuộc tính → bỏ __dict__ mỗi instance, giảm RAM & truy cập nhanh hơn. Hữu ích khi tạo hàng triệu object.
SENIOR
Idempotency là gì? Vì sao quan trọng?
ĐÁP ÁN
Thực hiện một thao tác nhiều lần cho kết quả như một lần (không gửi email 2 lần, không trừ tiền 2 lần). Quan trọng cho retry an toàn trong hệ phân tán — thường dùng idempotency key.
