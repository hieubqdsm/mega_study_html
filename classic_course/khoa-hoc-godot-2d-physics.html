L0 · Nhập môn Godot

CH 1 Godot là gì & cài đặt

Engine mã nguồn mở, nhẹ, cực hợp cho game 2D vật lý.

Godot là game engine mã nguồn mở (giấy phép MIT), làm được cả 2D & 3D. Điểm mạnh cho người mới: nhẹ (một file thực thi ~vài chục MB, không cần cài đặt), ngôn ngữ GDScript giống Python (dễ học), và kiến trúc node trực quan. Engine có sẵn physics 2D và các node lò xo/khớp nối — rất hợp mục tiêu Slinky của bạn.

Cài đặt

  1. Tải Godot 4.x từ godotengine.org/download (bản Standard dùng GDScript là đủ).
  2. Giải nén & chạy — không cần cài đặt.
  3. Project Manager → New Project → chọn renderer (Mobile/Compatibility hợp cho mobile) → Create.

CH 2 Scene, Node & cây cảnh

Tư duy "mọi thứ là node" — nền tảng kiến trúc Godot.

Trong Godot, mọi thứ là Node (sprite, va chạm, âm thanh, camera, body vật lý...). Các node ghép thành cây tạo nên một Scene. Một scene có thể nhúng (instance) vào scene khác — đây là cách tái sử dụng (vd: scene "Đốt Slinky" nhúng nhiều lần thành cả lò xo).

Signals — giao tiếp giữa node

Signal là cơ chế "phát tín hiệu" (observer pattern): một node phát, node khác lắng nghe — giúp các phần rời rạc mà vẫn liên lạc. Ví dụ nút bấm phát pressed, vùng va chạm phát body_entered.

Mẹo dàn cảnhTạo scene riêng cho thành phần lặp lại (player, viên đạn, đốt lò xo) rồi instance. Đổi scene gốc → mọi bản nhúng tự cập nhật. Đây là "prefab" của Godot.

CH 3 GDScript căn bản

Ngôn ngữ chính của Godot — giống Python, gắn vào node.

player.gd
extends Node2D          # script "kế thừa" loại node nó gắn vào

@export var speed: float = 200.0     # hiện trong Inspector, chỉnh không cần sửa code
@onready var sprite: Sprite2D = $Sprite2D  # lấy node con khi scene sẵn sàng

var health: int = 100
var velocity := Vector2.ZERO       # := tự suy kiểu

func _ready() -> void:        # gọi 1 lần khi node vào cây
    print("Sẵn sàng!")

func _process(delta: float) -> void:   # gọi MỖI khung hình; delta = thời gian giữa 2 frame
    position += velocity * delta

func take_damage(amount: int) -> void:
    health -= amount
    if health <= 0:
        queue_free()              # xóa node an toàn

CH 4 Game 2D đầu tiên

Di chuyển một sprite bằng phím/cảm ứng — nắm vòng lặp game.

Bước 1 — Input Map

Project → Project Settings → Input Map: thêm action move_left, move_right, move_up, move_down và gán phím (mũi tên/WASD). Dùng action thay vì hard-code phím để dễ đổi & hỗ trợ nhiều thiết bị.

Bước 2 — Scene & script

Tạo scene: Node2D (root) → thêm Sprite2D (gán một ảnh, vd icon.svg). Gắn script vào root:

mover.gd
extends Node2D

@export var speed: float = 300.0
@onready var sprite: Sprite2D = $Sprite2D

func _process(delta: float) -> void:
    # get_vector trả Vector2 đã chuẩn hóa từ 4 action (rất tiện)
    var dir := Input.get_vector("move_left", "move_right", "move_up", "move_down")
    sprite.position += dir * speed * delta

Nhấn F6 để chạy scene. Sprite di chuyển theo phím. 🎉

Input cho cảm ứngTrên mobile, dùng nút ảo (TouchScreenButton) gán cùng action, hoặc đọc InputEventScreenTouch/InputEventScreenDrag (ch9). Nhờ Input Map, cùng một action chạy cho cả phím lẫn cảm ứng.
✏️ Thực hành 4.1 dễ

Thêm giới hạn để sprite không ra khỏi màn hình (dùng clamp với kích thước viewport get_viewport_rect().size).

Gợi ý
var sz := get_viewport_rect().size
sprite.position.x = clamp(sprite.position.x, 0, sz.x)
sprite.position.y = clamp(sprite.position.y, 0, sz.y)

Godot có engine vật lý 2D tích hợp. Bạn dựng vật lý bằng các physics body + CollisionShape2D (mỗi body cần ít nhất một hình va chạm).

Collision Layers & Masks

Mỗi body có layer (nó "ở tầng nào") và mask (nó "quét va chạm với tầng nào"). Đây là cách kiểm soát cái gì va với cái gì — vd: đạn của người chơi chỉ trúng kẻ địch, không trúng người chơi.

Quy tắc nhớA phát hiện B khi mask của A bao gồm layer của B. Thường đặt tên layer (Project Settings → Layer Names → 2D Physics) để dễ quản lý.
Vật lý chạy ở _physics_processMọi thao tác lực/vận tốc/di chuyển body phải đặt trong _physics_process(delta) (timestep cố định, mặc định 60Hz) — không phải _process. Đặt sai chỗ → giật, thiếu ổn định, va chạm lỗi.

CH 6 RigidBody2D

Vật thể được engine mô phỏng — nền cho cách dựng Slinky thứ nhất.

RigidBody2D được engine tự lo: trọng lực, va đập, ma sát, nảy. Bạn không đặt position trực tiếp (sẽ "đánh nhau" với engine) — thay vào đó tác động bằng lực và xung lực (impulse).

ball.gd
extends RigidBody2D

func _ready():
    mass = 1.0
    gravity_scale = 1.0
    # nảy & ma sát đặt qua PhysicsMaterial trong Inspector (bounce, friction)

func _physics_process(delta):
    if Input.is_action_just_pressed("jump"):
        apply_central_impulse(Vector2.UP * 400)   # cú đẩy tức thời (nhảy/bắn)

func push_right():
    apply_central_force(Vector2.RIGHT * 600)        # lực liên tục mỗi bước
APITác dụng
apply_central_impulse(v)Xung lực tức thời tại tâm (nhảy, bắn, gõ)
apply_central_force(v)Lực liên tục (gió, lực đẩy) — gọi mỗi physics frame
apply_impulse(v, pos)Xung lực lệch tâm → tạo xoay
linear_velocity / angular_velocityĐọc/đặt vận tốc (đặt trực tiếp khi cần)
freezeTạm "đóng băng" body
PhysicsMaterialĐặt bounce (độ nảy 0–1) và friction qua resource PhysicsMaterial trên body (Inspector). Trọng lực toàn cục chỉnh ở Project Settings → Physics → 2D → Default Gravity.

CH 7 CharacterBody2D

Cho nhân vật bạn điều khiển trực tiếp — platformer/top-down.

Khác RigidBody (engine lo), CharacterBody2D để bạn tự tính vận tốc rồi gọi move_and_slide() — engine lo phần trượt dọc tường & phát hiện va chạm.

player.gd
extends CharacterBody2D

@export var speed: float = 300.0
@export var jump_force: float = 500.0
@export var gravity: float = 1200.0

func _physics_process(delta: float) -> void:
    # trọng lực
    if not is_on_floor():
        velocity.y += gravity * delta
    # nhảy
    if Input.is_action_just_pressed("jump") and is_on_floor():
        velocity.y = -jump_force
    # đi ngang
    var dir := Input.get_axis("move_left", "move_right")
    velocity.x = dir * speed
    move_and_slide()        # Godot 4: không truyền tham số; dùng thuộc tính velocity
Godot 4 vs 3Godot 3: KinematicBody2D + velocity = move_and_slide(velocity, Vector2.UP). Godot 4: CharacterBody2D có sẵn thuộc tính velocity, chỉ gọi move_and_slide(); dùng is_on_floor() để biết chạm đất.

CH 8 Area2D & phát hiện va chạm

Vùng cảm biến không cản — nhặt đồ, vùng sát thương, trigger.

Area2D phát hiện khi body/area khác đi vào/ra mà không cản chuyển động. Dùng signal:

extends Area2D

    if body.is_in_group("player"):
        collected.emit()      # phát signal

CH 9 Input chuột & cảm ứng — kéo thả

Tương tác trực tiếp: kéo vật bằng chuột/ngón tay (nền cho kéo Slinky).

draggable.gd
extends RigidBody2D

var dragging := false

func _ready():
    input_pickable = true      # cho phép body nhận input chuột (cần CollisionShape)
    input_event.connect(_on_input_event)

func _on_input_event(_viewport, event: InputEvent, _idx):
    if event is InputEventMouseButton and event.button_index == MOUSE_BUTTON_LEFT:
        dragging = event.pressed

func _physics_process(_delta):
    if dragging:
        # kéo "mềm": đặt vận tốc hướng về con trỏ (giữ ổn định vật lý)
        var target := get_global_mouse_position()
        linear_velocity = (target - global_position) * 20.0
Kéo "cứng" vs "mềm"Với RigidBody, đừng gán thẳng global_position = chuột (phá vật lý, nổ văng). Hãy kéo "mềm" bằng cách đặt linear_velocity hướng về con trỏ (như trên) — vật theo tay mà vẫn tương tác lò xo/va chạm mượt. Đây đúng là cách bạn sẽ kéo đốt Slinky.
Cảm ứngTrên mobile, bật Project Settings → Input Devices → Emulate Mouse From Touch để code chuột chạy luôn với cảm ứng; hoặc xử lý InputEventScreenTouch/InputEventScreenDrag để hỗ trợ đa chạm.

CH 10 Lý thuyết lò xo & dao động

Hiểu vật lý trước khi code — và 2 hướng mô phỏng Slinky.

Định luật Hooke & giảm chấn

Lực lò xo tỉ lệ với độ biến dạng, ngược chiều: kéo dài thì kéo về, nén thì đẩy ra. Thêm lực giảm chấn (damping) tỉ lệ vận tốc để dao động tắt dần (không rung mãi).

F_lò_xo = −k · x (k: độ cứng, x: độ lệch khỏi chiều dài nghỉ) F_giảm_chấn = −c · v (c: hệ số giảm chấn, v: vận tốc)

Slinky = chuỗi khối lượng nối bằng lò xo

Một Slinky là nhiều đốt (mass) nối liên tiếp bằng lò xo. Khi bạn lắc một đầu, lực truyền qua từng lò xo → tạo sóng lan truyền dọc thân (đặc trưng "uốn lượn" của slinky, và trò "đi xuống cầu thang"). Mở 🌀 Slinky Lab và bấm Lắc để thấy sóng này.

Verlet integration — vì sao ổn định

Thay vì lưu vận tốc, Verlet suy vận tốc từ vị trí trước đó: x_mới = x + (x − x_cũ)·damp + a·dt². Sau đó "kéo" các điểm về đúng khoảng cách nghỉ bằng ràng buộc (lặp vài lần). Cách "position-based" này rất ổn định với lò xo cứng — lý do nó là kỹ thuật kinh điển cho dây/vải/slinky.

CH 11 Joints trong Godot

Các khớp nối sẵn có để ràng buộc 2 body.

JointTác dụng

spring_setup.gd
var spring := DampedSpringJoint2D.new()
spring.node_a = body_a.get_path()      # 2 body cần nối
spring.node_b = body_b.get_path()
spring.length = 0                    # chiều dài tối đa (0 = tự)
spring.rest_length = 16             # chiều dài nghỉ của lò xo

CH 12 Slinky bằng RigidBody2D + DampedSpringJoint2D

Cách "engine-based": sinh chuỗi đốt & nối lò xo bằng code.

slinky_rigid.gd
extends Node2D

@export var segments: int = 18
@export var spacing: float = 16.0
@export var stiffness: float = 40.0
@export var damping: float = 1.5

var bodies: Array[RigidBody2D] = []

func _ready() -> void:
    var prev: PhysicsBody2D = null
    for i in segments:
        var seg := _make_segment(Vector2(0, i * spacing))
        # đốt đầu tiên cố định (treo lò xo lên)
        if i == 0:
            seg.freeze = true
            seg.freeze_mode = RigidBody2D.FREEZE_MODE_STATIC
        add_child(seg)
        bodies.append(seg)
        if prev:
            _connect_spring(prev, seg)
        prev = seg

func _make_segment(pos: Vector2) -> RigidBody2D:
    var b := RigidBody2D.new()
    b.position = pos
    b.mass = 0.4
    var shape := CollisionShape2D.new()
    var circle := CircleShape2D.new()
    circle.radius = 5.0
    shape.shape = circle
    b.add_child(shape)
    return b

func _connect_spring(a: PhysicsBody2D, b: PhysicsBody2D) -> void:
    var s := DampedSpringJoint2D.new()
    s.position = a.position
    s.node_a = a.get_path()
    s.node_b = b.get_path()
    s.rest_length = spacing
    s.stiffness = stiffness
    s.damping = damping
    add_child(s)

CH 13 Slinky bằng Verlet (tùy chỉnh)

Tự code mass-spring — đúng mô hình trong Slinky Lab. Ổn định & kiểm soát cao.

Ta lưu các điểm (vị trí + vị trí trước), tích phân Verlet với trọng lực, rồi áp ràng buộc khoảng cách nhiều lần để mô phỏng lò xo cứng. Điểm đầu được "ghim".

slinky_verlet.gd
extends Node2D

@export var segments: int = 20
@export var rest: float = 12.0      # khoảng cách nghỉ giữa 2 điểm
@export var stiffness: float = 0.6   # 0..1 (độ cứng ràng buộc)
@export var damping: float = 0.98    # giữ vận tốc (gần 1 = ít mất năng lượng)
@export var gravity: float = 900.0
@export var iterations: int = 6      # số lần lặp ràng buộc/khung (cao = cứng hơn)

var pos: PackedVector2Array = []      # vị trí hiện tại
var prev: PackedVector2Array = []     # vị trí khung trước
var pinned: Array[bool] = []          # điểm bị ghim?

func _ready() -> void:
    for i in segments:
        var p := Vector2(0, i * rest)
        pos.append(p); prev.append(p)
        pinned.append(i == 0)        # ghim điểm đầu

func _physics_process(delta: float) -> void:
    _integrate(delta)
    for i in iterations:
        _solve_constraints()
    queue_redraw()                    # vẽ lại (ch14)

func _integrate(delta: float) -> void:
    var g := Vector2(0, gravity) * delta * delta
    for i in pos.size():
        if pinned[i]:
            continue
        var velocity := (pos[i] - prev[i]) * damping
        prev[i] = pos[i]
        pos[i] += velocity + g

func _solve_constraints() -> void:
    for i in pos.size() - 1:
        var a := pos[i]
        var b := pos[i + 1]
        var delta := b - a
        var dist := delta.length()
        if dist == 0.0:
            continue
        var diff := (dist - rest) / dist * 0.5 * stiffness
        var offset := delta * diff
        if not pinned[i]:
            pos[i] += offset
        if not pinned[i + 1]:
            pos[i + 1] -= offset
Giống hệt Slinky LabMô hình trên chính là thứ chạy trong 🌀 Slinky Lab. Hãy mở Lab, chỉnh các slider (stiffness/damping/gravity/segments) để cảm nhận từng tham số làm gì, rồi mang đúng giá trị đó sang @export trong Godot.
Vì sao lặp ràng buộcMỗi lần _solve_constraints kéo các điểm gần đúng khoảng cách nghỉ hơn. Lặp nhiều lần (iterations) → lò xo "cứng" hơn & ổn định hơn. Đây là kỹ thuật Position-Based Dynamics.

CH 14 Vẽ & tạo hình coil

Biến chuỗi điểm thành hình lò xo xoắn đặc trưng.

Có 2 cách: dùng Line2D (đẩy điểm vào points) hoặc _draw() tùy chỉnh. Để có coil xoắn, ta vẽ đường tâm rồi lệch vuông góc theo hàm sin — pha sin chạy theo vị trí dọc thân nên khi nén/giãn, các vòng tự bó lại/giãn ra (đúng như slinky thật).

slinky_draw.gd (thêm vào script ch13)
@export var coils: int = 20      # số vòng xoắn
@export var coil_radius: float = 14.0
@export var color: Color = Color("6bb0e0")

func _draw() -> void:
    var n := pos.size() - 1
    if n < 1:
        return
    var path: PackedVector2Array = []
    var sub := 7
    for i in n:
        var a := pos[i]
        var b := pos[i + 1]
        var tangent := (b - a).normalized()
        var normal := Vector2(-tangent.y, tangent.x)   # vuông góc
        for s in sub:
            var f := float(s) / sub
            var t := (i + f) / float(n)             # 0..1 dọc toàn thân
            var center := a.lerp(b, f)
            var off := coil_radius * sin(t * coils * TAU)
            path.append(center + normal * off)
    path.append(pos[n])
    # vẽ đường nối các điểm coil
    for i in path.size() - 1:
        draw_line(path[i], path[i + 1], color, 3.0, true)
Line2D thay cho _drawKhông muốn tự _draw? Thêm node Line2D con, mỗi frame gán line.points = path (cùng cách tính coil), bật Antialiased, đặt width & gradient. Line2D còn hỗ trợ texture để slinky trông như kim loại.

    var best := -1
    var best_d := 400.0      # bán kính bắt điểm (px^2)
    for i in pos.size():
        var d := pos[i].distance_squared_to(m)
        if d < best_d:
            best_d = d; best = i
    return best

CH 16 Tinh chỉnh cảm giác (tuning)

Cùng mô hình, tham số khác nhau cho ra "cảm giác" rất khác.

Tham sốTăng lên thì...Gợi ý
iterations / stiffnessLò xo cứng hơn, ít giãnSlinky mềm: iterations thấp (3–5); dây cứng: cao (10+)
dampingGần 1 = nảy lâu; thấp = tắt nhanh0.96–0.99 cho slinky "sống"

Ổn định số học

  • Sub-stepping: nếu chuyển động nhanh gây "nổ", chia mỗi physics frame thành 2–4 bước nhỏ (lặp integrate+constraints).
  • Giới hạn vận tốc: kẹp độ dịch chuyển mỗi frame để tránh điểm văng xa.
  • Frame-rate: dùng _physics_process (timestep cố định) để hành vi nhất quán mọi máy.
  • stiffness ≤ 1 trong mô hình ràng buộc; muốn cứng hơn thì tăng iterations thay vì đẩy stiffness > 1 (gây dao động).

🌀 Bước 2 — Script lò xo (slinky.gd)

Mở rộng từ chương 13 (Verlet) + 14 (vẽ coil): thêm điểm treo đi theo một target, và hàm lấy đuôi.

slinky.gd
extends Node2D

@export var segments: int = 16
@export var rest: float = 14.0
@export var damping: float = 0.98
@export var gravity: float = 900.0
@export var iterations: int = 6
@export var coils: int = 16
@export var coil_radius: float = 12.0

var pos: PackedVector2Array = []
var prev: PackedVector2Array = []
var anchor: Vector2 = Vector2.ZERO   # vị trí đầu treo (main set mỗi frame)

func setup(start: Vector2) -> void:
    anchor = start
    pos.clear(); prev.clear()
    for i in segments:
        var p := start + Vector2(0, i * rest)
        pos.append(p); prev.append(p)

func _physics_process(delta: float) -> void:
    if pos.is_empty():
        return
    # tích phân Verlet
    var g := Vector2(0, gravity) * delta * delta
    for i in range(1, pos.size()):
        var v := (pos[i] - prev[i]) * damping
        prev[i] = pos[i]
        pos[i] += v + g
    # ràng buộc + ghim đầu treo vào anchor
    for it in iterations:
        pos[0] = anchor
        for i in pos.size() - 1:
            var d := pos[i + 1] - pos[i]
            var dist := d.length()
            if dist == 0.0: continue
            var off := d * ((dist - rest) / dist * 0.5)
            if i != 0: pos[i] += off
            pos[i + 1] -= off
    queue_redraw()

func get_tail() -> Vector2:
    return pos[pos.size() - 1] if not pos.is_empty() else anchor

func _draw() -> void:
    var n := pos.size() - 1
    if n < 1: return
    var path: PackedVector2Array = []
    var sub := 7
    for i in n:
        var a := pos[i]; var b := pos[i + 1]
        var tg := (b - a).normalized()
        var nm := Vector2(-tg.y, tg.x)
        for s in sub:
            var f := float(s) / sub
            var t := (i + f) / float(n)
            path.append(a.lerp(b, f) + nm * (coil_radius * sin(t * coils * TAU)))
    path.append(pos[n])
    for i in path.size() - 1:
        draw_line(path[i], path[i + 1], Color("6bb0e0"), 3.0, true)
    draw_circle(anchor, 6, Color("ffca28"))            # đầu treo
    draw_circle(pos[n], 6, Color("ff7043"))            # đuôi

🎯 Bước 3 — Game chính (main.gd): màn, input, thắng/thua

main.gd
extends Node2D

@onready var slinky: Node2D = $Slinky
@onready var level_label: Label = $HUD/LevelLabel
@onready var msg_label: Label = $HUD/MsgLabel

# mỗi màn: vị trí treo, đích (tâm+bán kính), danh sách chướng ngại (Rect2)
var levels := [
    {"anchor": Vector2(120, 60), "goal": Vector2(620, 340), "goal_r": 30.0,
     "obstacles": [Rect2(320, 150, 28, 240)]},
    {"anchor": Vector2(110, 60), "goal": Vector2(640, 160), "goal_r": 28.0,
     "obstacles": [Rect2(280, 0, 28, 230), Rect2(470, 200, 28, 280)]},
    {"anchor": Vector2(120, 60), "goal": Vector2(380, 420), "goal_r": 28.0,
     "obstacles": [Rect2(220, 130, 300, 26), Rect2(260, 280, 300, 26)]},
]
var current := 0
var dragging := false
var cleared := false

func _ready() -> void:
    $HUD/RetryButton.pressed.connect(load_level)
    $HUD/SkipButton.pressed.connect(_on_skip)
    load_level()

func load_level() -> void:
    var lv = levels[current]
    slinky.setup(lv["anchor"])
    cleared = false
    msg_label.text = ""
    level_label.text = "Màn %d / %d" % [current + 1, levels.size()]
    queue_redraw()

func _unhandled_input(event: InputEvent) -> void:
    if event is InputEventMouseButton and event.button_index == MOUSE_BUTTON_LEFT:
        dragging = event.pressed
    elif event is InputEventScreenTouch:
        dragging = event.pressed

func _physics_process(_delta: float) -> void:
    # kéo đầu treo theo con trỏ (kẹp trong màn)
    if dragging:
        var m := get_global_mouse_position()
        slinky.anchor = Vector2(clamp(m.x, 10, 780), clamp(m.y, 20, 440))
    if not cleared:
        _check()

func _check() -> void:
    var lv = levels[current]
    # thắng: đuôi chạm đích
    if slinky.get_tail().distance_to(lv["goal"]) < lv["goal_r"]:
        _win(); return
    # thua: BẤT KỲ đốt nào chạm chướng ngại
    for r in lv["obstacles"]:
        for p in slinky.pos:
            if r.has_point(p):
                _fail(); return

func _win() -> void:
    cleared = true
    msg_label.text = "🎉 Qua màn!"
    await get_tree().create_timer(0.8).timeout
    current = (current + 1) % levels.size()
    load_level()

func _fail() -> void:
    msg_label.text = "💥 Chạm gai! Thử lại"
    load_level()

func _on_skip() -> void:
    # xem quảng cáo rewarded để bỏ qua màn (ch24)
    if Engine.has_singleton("AdMob") or true:   # Ads là Autoload
        Ads.show_rewarded()      # khi xem xong, callback gọi skip_level()

func skip_level() -> void:    # Ads gọi hàm này sau khi thưởng
    current = (current + 1) % levels.size()
    load_level()

func _draw() -> void:
    var lv = levels[current]
    # chướng ngại
    for r in lv["obstacles"]:
        draw_rect(r, Color("f85149"))
    # đích
    draw_circle(lv["goal"], lv["goal_r"], Color("2ea043", 0.35))
    draw_arc(lv["goal"], lv["goal_r"], 0, TAU, 32, Color("2ea043"), 3.0)
Lưu ý nối dâyVì main.gd đọc slinky.pos và đặt slinky.anchor trực tiếp, hãy đảm bảo main._physics_process chạy cùng nhịp với slinky (cả hai đều trong _physics_process nên OK). Đích/chướng ngại vẽ ở main._draw; cần gọi queue_redraw() khi đổi màn (đã có trong load_level).

🆕 Bước 7 — Cơ chế puzzle: hộp đẩy + công tắc + cổng

Bản web (màn 3) đã có: hộp 🟫 đẩy được → đè lên công tắc 🟨 → mở cổng tím 🔒 chặn đường tới đích. Trong Godot:

  • Hộp: RigidBody2D + CollisionShape (chữ nhật) + Sprite. Sàn/tường là StaticBody2D → hộp tự rơi & nằm yên. Slinky (hình học) đẩy hộp bằng cách cấp xung lực khi điểm slinky chạm hộp.
  • Công tắc: Area2D — bật khi hộp (RigidBody) đi vào (body_entered).
  • Cổng: StaticBody2D (tường đặc) + visual; khi công tắc bật → tắt collision & ẩn đi.
box_push + switch + gate (thêm vào main.gd)
@onready var box: RigidBody2D = $Box
@onready var gate: StaticBody2D = $Gate
@onready var switch_area: Area2D = $Switch

func _ready() -> void:
    switch_area.body_entered.connect(_on_switch)

func _on_switch(body: Node) -> void:
    if body == box:
        gate.hide()
        gate.get_node("CollisionShape2D").set_deferred("disabled", true)  # mở cổng

func _push_boxes() -> void:    # gọi trong _physics_process
    var rect := Rect2(box.global_position - box_size * 0.5, box_size)
    for p in slinky.pos:
        if rect.has_point(p):
            var dir := signf(box.global_position.x - p.x)
            box.apply_central_impulse(Vector2(dir * 12, 0))  # slinky huých hộp
Vì sao đẩy bằng impulseSlinky của ta là hình học (Verlet), không phải physics body, nên không tự va chạm với RigidBody. Cách gọn: phát hiện điểm slinky nằm trong hộp → apply_central_impulse theo hướng đẩy. Muốn "thật" hơn nữa, có thể làm mỗi đốt là một RigidBody2D (cách ch12) — đổi lại nặng hơn.

⏱️ Bước 8 — Tính giờ, sao & lưu kỷ lục

Bản web đã có: đồng hồ, sao ⭐ theo thời gian (so với par), và lưu kỷ lục qua localStorage. Trong Godot dùng FileAccess + JSON ghi vào user://.

var elapsed := 0.0
var playing := false

func _physics_process(delta: float) -> void:
    if playing and not cleared:
        elapsed += delta
    # ... (kéo anchor, _check, _push_boxes) ...

func _stars_for(t: float) -> int:
    var par: float = levels[current].get("par", 10.0)
    if t <= par: return 3
    elif t <= par * 1.7: return 2
    return 1

func _win() -> void:
    cleared = true
    var stars := _stars_for(elapsed)
    msg_label.text = "🎉 Qua màn! " + "⭐".repeat(stars)
    _save_best(current, stars, elapsed)
    await get_tree().create_timer(0.9).timeout
    current = (current + 1) % levels.size(); load_level()

# ---- Save/Load: user://save.json ----
func _save_best(lvl: int, stars: int, t: float) -> void:
    var data := _load_all()
    var key := str(lvl)
    var old: Dictionary = data.get(key, {"stars": 0, "time": 9999.0})
    if stars > old["stars"] or (stars == old["stars"] and t < old["time"]):
        data[key] = {"stars": stars, "time": t}
        var f := FileAccess.open("user://save.json", FileAccess.WRITE)
        f.store_string(JSON.stringify(data)); f.close()

func _load_all() -> Dictionary:
    if not FileAccess.file_exists("user://save.json"): return {}
    var f := FileAccess.open("user://save.json", FileAccess.READ)
    var res = JSON.parse_string(f.get_as_text()); f.close()
    return res if res is Dictionary else {}
user:// là gìuser:// là thư mục lưu riêng của game (mỗi nền tảng một chỗ: AppData/Documents/...). An toàn để ghi save, settings, kỷ lục — tương đương localStorage của bản web.

ĐÁP ÁN
Suy vận tốc từ vị trí trước: x' = x + (x−x_cũ)·damp + a·dt², rồi áp ràng buộc khoảng cách. Rất ổn định cho lò xo/dây.
