Làm chủ LangChain — khung sườn xây ứng dụng LLM & RAG

16 chương theo 4 cấp độ. LangChain ghép các thành phần (model, prompt, parser, retriever, tool) thành chuỗi để xây chatbot, RAG, agent. Mỗi chương có code chạy được, bài tập tự chấm, quiz, flashcard. Đi từ prompt | model | parser → RAG → agent → LangGraph → triển khai.

Chuẩn bị Cài pip install langchain langchain-anthropic langchain-community langchain-chroma và đặt API key (vd ANTHROPIC_API_KEY). Ví dụ dùng Claude (claude-sonnet-4-6) nhưng đổi sang OpenAI/Gemini chỉ là thay lớp model. Bài tập tự chấm offline bằng cách kiểm tra "dấu hiệu" trong code bạn gõ.
L0 · Nhập môn

Chương 1 LangChain là gì & cài đặt

Vì sao cần một "khung sườn" cho ứng dụng LLM.

LangChain là framework giúp ghép LLM với dữ liệu & công cụ bên ngoài thành ứng dụng thật: chatbot có kiến thức riêng (RAG), agent biết gọi tool, pipeline xử lý nhiều bước. Thay vì tự viết glue code, bạn lắp ráp các thành phần chuẩn hóa.

GóiVai trò
langchain-coreTrừu tượng nền (Runnable, messages, prompts)
langchainChains, agents, retrieval cấp cao
langchain-anthropic / -openaiTích hợp nhà cung cấp model
langchain-communityLoader, vector store, tool cộng đồng
LangGraphĐồ thị trạng thái cho agent phức tạp
LangSmithTheo dõi, debug, đánh giá (tracing)

1. LangChain chủ yếu giải quyết việc gì?

Chương 2 Chat models & messages

Đơn vị giao tiếp cơ bản với LLM.
python●
from langchain_anthropic import ChatAnthropic
from langchain_core.messages import SystemMessage, HumanMessage

llm = ChatAnthropic(model="claude-sonnet-4-6", temperature=0)

resp = llm.invoke([
    SystemMessage("Bạn là trợ lý súc tích."),
    HumanMessage("Thủ đô Việt Nam là gì?"),
])
print(resp.content)        # "Hà Nội."
3 loại message SystemMessage (đặt vai trò/luật), HumanMessage (người dùng), AIMessage (model trả lời). invoke nhận list message (hoặc 1 chuỗi) → trả về AIMessage; lấy nội dung qua .content.
✍️ Bài tập 1 — Gọi model
Khởi tạo ChatAnthropic(model="claude-sonnet-4-6") rồi .invoke("Xin chào"), in .content.
Xem lời giải
llm = ChatAnthropic(model="claude-sonnet-4-6")
resp = llm.invoke("Xin chào")
print(resp.content)

Chương 3 Prompt templates

Tách prompt khỏi code, điền biến lúc chạy.
python●
from langchain_core.prompts import ChatPromptTemplate

prompt = ChatPromptTemplate.from_messages([
    ("system", "Bạn là chuyên gia {linh_vuc}."),
    ("human", "Giải thích {khai_niem} trong 2 câu."),
])
msgs = prompt.invoke({"linh_vuc": "vật lý", "khai_niem": "entropy"})
Vì sao dùng template Tái sử dụng, dễ test, tránh nối chuỗi thủ công. {biến} được điền qua invoke(dict). Template là một Runnable → ghép thẳng vào chuỗi (chương 5).
✍️ Bài tập 2 — Tạo prompt template
Tạo ChatPromptTemplate.from_messages với 1 dòng "system" và 1 dòng "human" chứa biến {cau_hoi}.
Xem lời giải
prompt = ChatPromptTemplate.from_messages([
    ("system", "Bạn là trợ lý hữu ích."),
    ("human", "{cau_hoi}"),
])
L1 · LCEL & Chains

Chương 4 Output parsers

Biến text thô của LLM thành dữ liệu dùng được.
python●
from langchain_core.output_parsers import StrOutputParser
parser = StrOutputParser()          # AIMessage → str

# Structured output: ép model trả đúng schema (Pydantic)
from pydantic import BaseModel
class Phim(BaseModel):
    ten: str
    nam: int
structured = llm.with_structured_output(Phim)
structured.invoke("Phim Inception ra năm nào?")   # Phim(ten='Inception', nam=2010)
with_structured_output Cách hiện đại & tin cậy nhất để lấy JSON/đối tượng từ LLM — model dùng tool-calling để điền đúng schema, không cần parse chuỗi mong manh.
✍️ Bài tập 3 — Structured output
Định nghĩa Pydantic class Sach(BaseModel) với tieu_de: str; tạo llm.with_structured_output(Sach).
Xem lời giải
class Sach(BaseModel):
    tieu_de: str
structured = llm.with_structured_output(Sach)

Chương 5 LCEL: nối bằng |

LangChain Expression Language — trái tim của LangChain hiện đại.

Mọi thành phần là một Runnable và nối được bằng toán tử | (như pipe trong shell). Output của bước này là input của bước sau.

python · chuỗi kinh điển●
chain = prompt | llm | StrOutputParser()

chain.invoke({"linh_vuc": "sinh học", "khai_niem": "ADN"})
# prompt điền biến → llm sinh → parser lấy text. Trả về str.

chain.batch([{...}, {...}])     # chạy song song nhiều input
for chunk in chain.stream({...}): print(chunk, end="")   # streaming
3 method chuẩn của mọi Runnable invoke (1 input), batch (nhiều input song song), stream (trả từng phần). Ghép bằng | tự động truyền dữ liệu.
✍️ Bài tập 4 — Dựng chuỗi LCEL
Nối prompt, llm, StrOutputParser() thành chain bằng toán tử |.
Xem lời giải
chain = prompt | llm | StrOutputParser()

Chương 6 Runnable & rẽ nhánh

Truyền song song, biến đổi, chèn dữ liệu vào chuỗi.
python●
from langchain_core.runnables import RunnablePassthrough, RunnableLambda

# Chạy song song & gom thành dict (rất hay dùng trong RAG)
chain = (
    {"context": retriever, "question": RunnablePassthrough()}
    | prompt | llm | StrOutputParser()
)

upper = RunnableLambda(lambda x: x.upper())   # bọc hàm Python bất kỳ
RunnablePassthrough chuyển input nguyên vẹn sang nhánh — kết hợp với một dict để vừa giữ câu hỏi gốc vừa thêm context truy hồi. Đây là khuôn mẫu RAG bạn sẽ gặp ở chương 11.

Chương 7 Memory / lịch sử hội thoại

Cho chatbot "nhớ" các lượt trước.
python●
from langchain_core.prompts import MessagesPlaceholder
from langchain_core.chat_history import InMemoryChatMessageHistory
from langchain_core.runnables.history import RunnableWithMessageHistory

prompt = ChatPromptTemplate.from_messages([
    ("system", "Bạn là trợ lý."),
    MessagesPlaceholder("history"),     # chèn lịch sử vào đây
    ("human", "{input}"),
])
store = {}
get_hist = lambda sid: store.setdefault(sid, InMemoryChatMessageHistory())
chat = RunnableWithMessageHistory(prompt | llm, get_hist,
        input_messages_key="input", history_messages_key="history")
chat.invoke({"input": "Tôi tên An"}, config={"configurable": {"session_id": "u1"}})
MessagesPlaceholder là "chỗ trống" để LangChain tự chèn toàn bộ lịch sử hội thoại theo session_id. Mỗi phiên có history riêng.
L2 · RAG (Retrieval-Augmented Generation)

Chương 8 Loader & text splitter

Nạp tài liệu & cắt thành đoạn vừa ngữ cảnh.
python●
from langchain_community.document_loaders import PyPDFLoader
from langchain_text_splitters import RecursiveCharacterTextSplitter

docs = PyPDFLoader("tai-lieu.pdf").load()    # list[Document]
splitter = RecursiveCharacterTextSplitter(chunk_size=1000, chunk_overlap=200)
chunks = splitter.split_documents(docs)
chunk_size & overlap Đoạn quá lớn → loãng ngữ cảnh & tốn token; quá nhỏ → mất mạch. chunk_overlap giữ vài câu gối đầu để không cắt mất ý ở ranh giới. Khởi đầu hợp lý: 1000 / 200.
✍️ Bài tập 5 — Cắt văn bản
Tạo RecursiveCharacterTextSplitter với chunk_size=500, chunk_overlap=50, rồi .split_documents(docs).
Xem lời giải
splitter = RecursiveCharacterTextSplitter(chunk_size=500, chunk_overlap=50)
chunks = splitter.split_documents(docs)

Chương 9 Embeddings & vector store

Biến văn bản thành vector để tìm theo ngữ nghĩa.
python●
from langchain_openai import OpenAIEmbeddings
from langchain_chroma import Chroma

emb = OpenAIEmbeddings()       # hoặc HuggingFaceEmbeddings (offline)
vectorstore = Chroma.from_documents(chunks, embedding=emb)
vectorstore.similarity_search("câu hỏi", k=4)   # 4 đoạn gần nhất
Embedding = "tọa độ ngữ nghĩa" Câu gần nghĩa → vector gần nhau. Vector store (Chroma, FAISS, pgvector...) lưu vector & tìm "hàng xóm gần nhất" cực nhanh. Đây là bộ nhớ tri thức của RAG.

Chương 10 Retriever

Giao diện chuẩn để "lấy tài liệu liên quan".
python●
retriever = vectorstore.as_retriever(search_kwargs={"k": 4})
docs = retriever.invoke("LangChain là gì?")    # list[Document]
Retriever là Runnable Nên ghép thẳng vào chuỗi LCEL bằng |. Có nhiều loại: vector, BM25 (từ khóa), MultiQuery, hoặc kết hợp (hybrid) để tăng độ phủ.
✍️ Bài tập 6 — Tạo retriever
Từ vectorstore, tạo retriever = vectorstore.as_retriever(search_kwargs={"k": 3}).
Xem lời giải
retriever = vectorstore.as_retriever(search_kwargs={"k": 3})

Chương 11 RAG chain hoàn chỉnh

Ghép mọi mảnh: hỏi → truy hồi → trả lời có dẫn nguồn.
python · RAG bằng LCEL●
from langchain_core.runnables import RunnablePassthrough
from langchain_core.output_parsers import StrOutputParser

prompt = ChatPromptTemplate.from_messages([
    ("system", "Trả lời CHỈ dựa vào ngữ cảnh:\n\n{context}"),
    ("human", "{question}"),
])
def format_docs(docs): return "\n\n".join(d.page_content for d in docs)

rag = (
    {"context": retriever | format_docs, "question": RunnablePassthrough()}
    | prompt | llm | StrOutputParser()
)
rag.invoke("Chính sách bảo hành bao lâu?")
Khuôn mẫu RAG (1) câu hỏi vào cả 2 nhánh; (2) nhánh context = retriever lấy đoạn rồi format; (3) prompt nhồi context + câu hỏi; (4) llm trả lời. Buộc "chỉ dựa vào ngữ cảnh" để giảm bịa (hallucination).

Chương 12 Tools & function calling

Cho LLM gọi hàm Python để làm việc thật.
python●
from langchain_core.tools import tool

@tool
def cong(a: int, b: int) -> int:
    """Cộng hai số nguyên."""     # docstring = mô tả cho LLM
    return a + b

llm_tools = llm.bind_tools([cong])           # cho model "biết" tool
ai = llm_tools.invoke("2 cộng 3 bằng mấy?")
print(ai.tool_calls)   # [{'name': 'cong', 'args': {'a': 2, 'b': 3}, ...}]
Tool = hàm + mô tả Decorator @tool biến hàm Python thành tool; LLM đọc tên + docstring + kiểu tham số để quyết định khi nào & gọi với args gì. Bạn vẫn phải thực thi tool rồi đưa kết quả lại cho model (agent ở chương 13 tự động hóa vòng này).
✍️ Bài tập 7 — Định nghĩa tool
Dùng decorator @tool cho hàm def nhan(a: int, b: int) -> int có docstring; rồi llm.bind_tools([nhan]).
Xem lời giải
@tool
def nhan(a: int, b: int) -> int:
    """Nhân hai số nguyên."""
    return a * b
llm_tools = llm.bind_tools([nhan])
L3 · Agents & triển khai

Chương 13 Agents

LLM tự quyết định gọi tool nào, lặp đến khi xong.
python · agent bằng LangGraph●
from langgraph.prebuilt import create_react_agent

agent = create_react_agent(llm, tools=[cong, nhan, search_tool])
result = agent.invoke({"messages": [("user", "Giá BTC giờ rồi nhân 2?")]})
print(result["messages"][-1].content)
Vòng lặp ReAct Agent = LLM + tools + vòng lặp: model suy luận → gọi tool → đọc kết quả → suy luận tiếp → ... → trả lời cuối. create_react_agent (LangGraph) là cách chuẩn hiện nay thay cho AgentExecutor cũ.
✍️ Bài tập 8 — Tạo agent
Dùng create_react_agent(llm, tools=[cong, nhan]) rồi .invoke({"messages": [("user", "...")]}).
Xem lời giải
agent = create_react_agent(llm, tools=[cong, nhan])
result = agent.invoke({"messages": [("user", "2 cộng 3 rồi nhân 4?")]})

Chương 14 LangGraph

Khi agent cần trạng thái, vòng lặp & phân nhánh có kiểm soát.
python●
from langgraph.graph import StateGraph, START, END
from typing import TypedDict

class State(TypedDict):
    input: str
    output: str

def node_llm(state): return {"output": llm.invoke(state["input"]).content}

g = StateGraph(State)
g.add_node("llm", node_llm)
g.add_edge(START, "llm"); g.add_edge("llm", END)
app = g.compile()
Vì sao LangGraph LCEL là chuỗi thẳng; LangGraph là đồ thị có trạng thái, cho phép vòng lặp, điều kiện, human-in-the-loop, checkpoint — cần thiết cho agent sản xuất phức tạp.

Chương 15 Streaming & LangSmith

Trải nghiệm mượt & quan sát được.
python●
for chunk in chain.stream({"question": "..."}):
    print(chunk, end="", flush=True)   # in dần như ChatGPT

# LangSmith: chỉ cần đặt biến môi trường là có trace đầy đủ

LangSmith Bật tracing để xem từng bước chuỗi/agent: prompt thực tế, token, độ trễ, chi phí, lỗi. Cực kỳ quan trọng khi debug RAG "trả lời sai" — thường do retriever lấy nhầm đoạn.

Chương 16 Dự án: RAG chatbot trên tài liệu của bạn

Ghép toàn bộ thành ứng dụng hỏi-đáp tài liệu.
  1. Nạp: DirectoryLoader/PyPDFLoader → RecursiveCharacterTextSplitter.
  2. Index: Chroma.from_documents(chunks, emb) (lưu xuống đĩa với persist_directory).
  3. Retriever: as_retriever(k=4) (cân nhắc hybrid + rerank).
  4. RAG chain: context+question → prompt → llm → parser; buộc dẫn nguồn.
  5. Lịch sử: bọc bằng RunnableWithMessageHistory cho hội thoại nhiều lượt.
  6. UI & quan sát: Streamlit/FastAPI + stream + LangSmith tracing.
✍️ Bài tập 9 — Mảnh ghép RAG
Dựng chuỗi: {"context": retriever, "question": RunnablePassthrough()} | prompt | llm | StrOutputParser().
Xem lời giải
rag = (
    {"context": retriever, "question": RunnablePassthrough()}
    | prompt | llm | StrOutputParser()
)
