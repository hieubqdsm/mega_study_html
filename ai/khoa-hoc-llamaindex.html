L0 · Nhập môn

Chương 1 LlamaIndex là gì & cài đặt

Khung dữ liệu chuyên cho RAG.

LlamaIndex là framework tập trung vào dữ liệu cho LLM: nạp tài liệu từ trăm nguồn, đánh chỉ mục (index) để tìm theo ngữ nghĩa, rồi truy vấn bằng ngôn ngữ tự nhiên. So với LangChain (đa năng, dàn dựng), LlamaIndex chuyên sâu & tối giản cho RAG — thường ít dòng code hơn cho cùng một pipeline hỏi-đáp tài liệu.

Khái niệmVai trò
Reader / LoaderNạp dữ liệu (PDF, web, Notion, SQL...) → Document
NodeĐoạn nhỏ (chunk) của Document, đơn vị truy hồi
IndexCấu trúc tra cứu (VectorStoreIndex, SummaryIndex...)
RetrieverLấy Node liên quan đến câu hỏi
Query / Chat engineGiao diện hỏi-đáp end-to-end
📝 Quiz nhanh
1. LlamaIndex mạnh nhất ở mảng nào?
✅ LlamaIndex là "data framework": nạp → index → truy vấn dữ liệu của bạn bằng LLM.

Chương 2 Document & Node

Hai đơn vị dữ liệu nền tảng.
python●
from llama_index.core import Document

doc = Document(
    text="LlamaIndex giúp xây RAG nhanh chóng.",
    metadata={"nguon": "sotay.pdf", "trang": 1},
)
# Document được cắt thành các Node khi đánh index
Document → Node Một Document là cả tài liệu/trang; khi index, nó được node parser cắt thành các Node (đoạn nhỏ kèm metadata & quan hệ). Node là thứ thực sự được nhúng & truy hồi.
✍️ Bài tập 1 — Tạo Document
Tạo một Document với text="Xin chào" và metadata={"nguon": "test"}.
Xem lời giải
doc = Document(text="Xin chào", metadata={"nguon": "test"})

Chương 3 Settings (LLM & embedding)

Cấu hình toàn cục: dùng model nào.
python●
from llama_index.core import Settings
from llama_index.llms.anthropic import Anthropic
from llama_index.embeddings.openai import OpenAIEmbedding

Settings.llm = Anthropic(model="claude-sonnet-4-6")
Settings.embed_model = OpenAIEmbedding(model="text-embedding-3-small")
Settings.chunk_size = 512
Settings = mặc định toàn cục Đặt một lần, mọi index/engine dùng theo. Vẫn có thể ghi đè cục bộ cho từng đối tượng. Embedding model quyết định chất lượng truy hồi; LLM quyết định chất lượng câu trả lời.
✍️ Bài tập 2 — Cấu hình Settings
Đặt Settings.llm = Anthropic(model="claude-sonnet-4-6") và Settings.chunk_size = 512.
Xem lời giải
Settings.llm = Anthropic(model="claude-sonnet-4-6")
Settings.chunk_size = 512
L1 · RAG cơ bản

Chương 4 VectorStoreIndex — RAG trong 5 dòng

Sức mạnh "tối giản" của LlamaIndex.
python · RAG tối thiểu●
from llama_index.core import VectorStoreIndex, SimpleDirectoryReader

documents = SimpleDirectoryReader("data").load_data()   # nạp cả thư mục
index = VectorStoreIndex.from_documents(documents)        # cắt + nhúng + lưu
query_engine = index.as_query_engine()
print(query_engine.query("Tài liệu nói gì về bảo hành?"))
5 dòng = RAG hoàn chỉnh from_documents tự lo cắt Node, gọi embedding, dựng vector index. as_query_engine gói luôn retriever + synthesizer. Đây là lý do LlamaIndex nổi tiếng "RAG nhanh".
✍️ Bài tập 3 — Dựng index
Tạo index = VectorStoreIndex.from_documents(documents).
Xem lời giải
index = VectorStoreIndex.from_documents(documents)

Chương 5 Query engine

Giao diện hỏi-đáp một lượt.
python●
qe = index.as_query_engine(similarity_top_k=4)   # lấy 4 Node liên quan
resp = qe.query("Chính sách đổi trả?")
print(resp)                  # câu trả lời
print(resp.source_nodes)     # các Node nguồn (để dẫn nguồn/kiểm chứng)
source_nodes = dẫn nguồn Luôn kiểm resp.source_nodes để biết câu trả lời dựa trên đoạn nào — vừa minh bạch, vừa giúp debug khi model trả lời sai (thường do truy hồi nhầm).
✍️ Bài tập 4 — Query engine
Tạo query_engine = index.as_query_engine(similarity_top_k=3) rồi gọi .query("...").
Xem lời giải
query_engine = index.as_query_engine(similarity_top_k=3)

Chương 7 Lưu & nạp lại index

Đừng nhúng lại mỗi lần chạy — tốn tiền & thời gian.
python●
from llama_index.core import StorageContext, load_index_from_storage

index.storage_context.persist(persist_dir="./storage")   # lưu xuống đĩa

# Lần sau: nạp lại, KHÔNG cần embedding lại
ctx = StorageContext.from_defaults(persist_dir="./storage")
index = load_index_from_storage(ctx)
Tiết kiệm chi phí Mỗi lần from_documents đều gọi API embedding (mất tiền). Production nên dùng vector DB thật (Chroma, Qdrant, pgvector) qua StorageContext thay vì lưu file mặc định.

Chương 8 Node parser / chunking

Cắt thông minh = truy hồi tốt.
python●
from llama_index.core.node_parser import SentenceSplitter

splitter = SentenceSplitter(chunk_size=512, chunk_overlap=50)
nodes = splitter.get_nodes_from_documents(documents)
index = VectorStoreIndex(nodes)        # dựng index từ Node có sẵn
Các parser hữu ích SentenceSplitter (cắt theo câu, mặc định tốt), SentenceWindowNodeParser (truy hồi câu + ngữ cảnh xung quanh), SemanticSplitter (cắt theo điểm đổi chủ đề), MarkdownNodeParser (theo cấu trúc heading).
✍️ Bài tập 6 — Cắt Node
Tạo SentenceSplitter(chunk_size=256, chunk_overlap=20) rồi .get_nodes_from_documents(documents).
Xem lời giải
splitter = SentenceSplitter(chunk_size=256, chunk_overlap=20)
nodes = splitter.get_nodes_from_documents(documents)

Chương 9 Retriever & chế độ

Tách riêng bước "lấy tài liệu" để tùy biến.
python●
retriever = index.as_retriever(similarity_top_k=5)
nodes = retriever.retrieve("câu hỏi")     # list[NodeWithScore]

# Tự lắp query engine từ retriever + postprocessor
from llama_index.core.query_engine import RetrieverQueryEngine
from llama_index.core.postprocessor import SimilarityPostprocessor
qe = RetrieverQueryEngine.from_args(
    retriever, node_postprocessors=[SimilarityPostprocessor(similarity_cutoff=0.7)])
Postprocessor Lọc/sắp xếp lại Node sau truy hồi: cắt theo ngưỡng điểm, rerank bằng model chuyên dụng (Cohere/LLM), hay mở rộng ngữ cảnh. Rerank thường tăng chất lượng RAG đáng kể.

Chương 10 Response synthesizer

Cách "tổng hợp" câu trả lời từ nhiều Node.
response_modeCách hoạt độngDùng khi

Chương 11 Metadata & lọc

Truy vấn chính xác hơn nhờ siêu dữ liệu.
python●
from llama_index.core.vector_stores import MetadataFilters, ExactMatchFilter

filters = MetadataFilters(filters=[ExactMatchFilter(key="nam", value=2026)])
qe = index.as_query_engine(filters=filters)   # chỉ tìm trong tài liệu năm 2026
Kết hợp vector + metadata Lọc theo metadata (tác giả, năm, phòng ban, quyền truy cập) trước rồi mới tìm ngữ nghĩa → vừa chính xác vừa an toàn (phân quyền). Gắn metadata giàu khi nạp Document để khai thác sau.

Chương 12 Chat engine

RAG có trí nhớ hội thoại nhiều lượt.
python●
chat_engine = index.as_chat_engine(chat_mode="condense_plus_context")
print(chat_engine.chat("Sản phẩm bảo hành bao lâu?"))
print(chat_engine.chat("Thế còn đổi trả?"))   # hiểu "thế còn" nhờ nhớ lượt trước
Query vs Chat engine query_engine = hỏi-đáp một lượt (không nhớ). chat_engine = có lịch sử, biết "viết lại" câu hỏi theo ngữ cảnh hội thoại (condense_plus_context là chế độ cân bằng tốt cho RAG).
✍️ Bài tập 7 — Chat engine
Tạo chat_engine = index.as_chat_engine(chat_mode="condense_plus_context") rồi .chat("...").
Xem lời giải
chat_engine = index.as_chat_engine(chat_mode="condense_plus_context")

Chương 13 Agents & tools

LLM tự chọn tool — kể cả "tool là một query engine".
python●
from llama_index.core.tools import FunctionTool, QueryEngineTool
from llama_index.core.agent import ReActAgent

def nhan(a: int, b: int) -> int:
    """Nhân hai số."""
    return a * b

tools = [
    FunctionTool.from_defaults(fn=nhan),
    QueryEngineTool.from_defaults(query_engine, name="tai_lieu",
        description="Trả lời câu hỏi về tài liệu công ty"),
]
agent = ReActAgent.from_tools(tools, verbose=True)
agent.chat("Bảo hành mấy năm, nhân với 12 ra mấy tháng?")
QueryEngineTool Biến một RAG index thành một tool để agent gọi khi cần tra cứu — nền tảng của multi-document agent (mỗi tài liệu/chủ đề một query engine riêng, agent định tuyến).
✍️ Bài tập 8 — Tạo tool & agent
Tạo FunctionTool.from_defaults(fn=nhan) rồi ReActAgent.from_tools(tools).
Xem lời giải
tools = [FunctionTool.from_defaults(fn=nhan)]
agent = ReActAgent.from_tools(tools, verbose=True)

Chương 14 Workflows

Pipeline phức tạp theo sự kiện (event-driven).
python●
from llama_index.core.workflow import Workflow, step, StartEvent, StopEvent

class RAGFlow(Workflow):
    @step
    async def run(self, ev: StartEvent) -> StopEvent:
        ans = query_engine.query(ev.query)
        return StopEvent(result=str(ans))
Workflows Cách hiện đại của LlamaIndex để dựng pipeline nhiều bước có rẽ nhánh/vòng lặp (thay cho Query Pipeline cũ). Mỗi @step nhận & phát ra event; phù hợp cho RAG nâng cao, agent tự sửa, multi-step reasoning.

Chương 15 Đánh giá & quan sát

RAG "có vẻ đúng" ≠ đúng. Phải đo.
python●
from llama_index.core.evaluation import FaithfulnessEvaluator, RelevancyEvaluator

faith = FaithfulnessEvaluator()      # trả lời có bám nguồn không (chống bịa)?
rel = RelevancyEvaluator()           # trả lời có liên quan câu hỏi không?
result = faith.evaluate_response(response=qe.query("..."))
print(result.passing)
Đo gìCông cụ
Bịa đặt (hallucination)FaithfulnessEvaluator
Độ liên quanRelevancyEvaluator
Chất lượng truy hồihit-rate, MRR (RetrieverEvaluator)
Quan sát/tracingLlamaTrace, Arize Phoenix, LangSmith

Chương 16 Dự án thực chiến: Hỏi-đáp tài liệu công ty

Ghép toàn bộ thành trợ lý nội bộ.
  1. Nạp: SimpleDirectoryReader("docs", recursive=True) (PDF/docx/md).
  2. Cắt: SentenceSplitter + gắn metadata (phòng ban, ngày).
  3. Index + lưu: VectorStoreIndex vào vector DB (Qdrant), persist.
  4. Truy vấn: retriever (top_k) + rerank + response_mode phù hợp; lọc metadata theo quyền.
  5. Hội thoại: as_chat_engine để hỏi nhiều lượt; trả kèm source_nodes.
  6. Đánh giá & UI: Faithfulness/Relevancy + Streamlit + tracing.
✍️ Bài tập 9 — RAG end-to-end
Ghép 3 dòng: load_data() → VectorStoreIndex.from_documents(...) → as_query_engine().query("...").
Xem lời giải
documents = SimpleDirectoryReader("data").load_data()
index = VectorStoreIndex.from_documents(documents)
resp = index.as_query_engine().query("Câu hỏi của bạn?")
Hoàn thành khóa học 🎉 Bạn đã đi từ index 5 dòng đến RAG production có agent & đánh giá. Bước tiếp: thử rerank, multi-document agent, và đối chiếu khóa LangChain — nhiều dự án dùng cả hai: LlamaIndex cho tầng dữ liệu/RAG, LangChain/LangGraph cho dàn dựng agent.
