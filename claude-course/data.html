Module 4

Xử lý Data với Claude

01 Hai chế độ làm việc với dữ liệu

Trước khi làm gì, hãy hiểu sự khác biệt — nó quyết định độ tin cậy của kết quả.

📊 Chế độ A — Claude tự "đọc" dữ liệu

Bạn dán bảng hoặc tải file lên, Claude đọc và trả lời trực tiếp. Nhanh, hợp dữ liệu nhỏ. Nhưng với phép tính trên nhiều dòng, nó ước lượng — có thể lệch.

⚙️ Chế độ B — Claude viết code xử lý

Claude viết đoạn code (Python/SQL) rồi chạy để tính. Con số là kết quả tính toán thật, lặp lại được, kiểm tra được. Hợp dữ liệu lớn và mọi việc cần độ chính xác.

🎯

Quy tắc vàng của module này: với bất kỳ con số quan trọng nào, hãy để Claude viết code và chạy — đừng để nó "nhẩm". Trên Claude.ai dùng tính năng phân tích/chạy code; trong Claude Code thì nó chạy script trực tiếp. "Tổng doanh thu là bao nhiêu" phải đến từ một phép tính, không phải một phỏng đoán.

Phần còn lại của module nói rõ khi nào dùng chế độ nào. Mặc định: dữ liệu nhỏ + câu hỏi định tính → chế độ A; mọi thứ cần con số đúng → chế độ B.

02 Làm sạch & chuẩn hóa dữ liệu

Dữ liệu thực tế luôn lộn xộn. Đây là việc tẻ nhạt nhất — và Claude xử lý rất tốt.

Khảo sát trước khi sửa

Prompt — chẩn đoán dữ liệu
Đây là dữ liệu của tôi [đính kèm file / dán mẫu vài chục dòng].

Hãy viết code khảo sát và cho tôi báo cáo chất lượng dữ liệu:
- Mỗi cột: kiểu dữ liệu, số giá trị thiếu, số giá trị duy nhất.
- Vấn đề về định dạng: ngày tháng không nhất quán, số bị lưu dạng
  chữ, khoảng trắng thừa, hoa/thường lẫn lộn.
- Dòng trùng lặp.
- Giá trị bất thường đáng ngờ.
- Đánh giá: dữ liệu này có thể tin tới mức nào?

Chỉ báo cáo — CHƯA sửa gì. Tôi sẽ quyết cách xử lý từng vấn đề.

Làm sạch theo từng quyết định

Đừng nói "làm sạch giúp tôi" — làm sạch là một chuỗi quyết định. Dòng thiếu giá trị: xóa, hay điền? Điền bằng gì? Hãy quyết, rồi để Claude thực thi:

Prompt — làm sạch dữ liệu
Làm sạch dữ liệu theo đúng các quyết định sau:
- Cột ngày: chuẩn hóa hết về YYYY-MM-DD.
- Email thiếu: giữ dòng, đánh dấu cột là "chưa có".
- Dòng trùng: gộp, giữ bản ghi mới nhất.
- Cột [tên]: bỏ khoảng trắng thừa, đưa hết về chữ thường.

Viết code làm sạch. Sau khi chạy:
- Báo từng bước đã đổi bao nhiêu dòng.
- Xuất file mới, GIỮ NGUYÊN file gốc.
- Liệt kê những gì bạn KHÔNG chắc và đã bỏ qua.
⚠️

Luôn yêu cầu giữ nguyên file gốc và xuất ra file mới. Làm sạch dữ liệu có thể sai; bạn cần đường lùi. Và luôn bắt Claude báo cáo đã thay đổi gì — "im lặng làm sạch" che mất lỗi.

03 Phân tích CSV / Excel

Trả lời câu hỏi kinh doanh từ một bảng tính — mà không cần biết viết công thức.

Hỏi bằng ngôn ngữ thường, nhận lại con số có kiểm chứng

Prompt — phân tích bảng tính
Đính kèm: [file dữ liệu của bạn].

Trả lời các câu hỏi sau bằng cách VIẾT CODE và CHẠY — không ước lượng:
1. [câu hỏi 1 — vd: doanh thu mỗi tháng năm nay]
2. [câu hỏi 2 — vd: 10 khách hàng đóng góp doanh thu lớn nhất]
3. [câu hỏi 3 — vd: nhóm sản phẩm nào đang tăng/giảm]

Với mỗi câu: nêu kết quả, cho xem đoạn code đã dùng, và nói rõ
mọi giả định (vd: cách định nghĩa "khách hàng đang hoạt động").

Để Claude tự tìm điều đáng chú ý

Đôi khi bạn chưa biết nên hỏi gì. Hãy mở rộng — nhưng vẫn neo vào dữ liệu thật:

Prompt — khám phá dữ liệu
Đây là dữ liệu [mô tả: nó là gì, mỗi dòng đại diện cho cái gì].

Bối cảnh: tôi quan tâm tới [mục tiêu kinh doanh].

Hãy khám phá dữ liệu (viết code, chạy thật) và cho tôi 5 phát hiện
đáng chú ý nhất liên quan tới mục tiêu trên. Mỗi phát hiện:
- Nói rõ nó là gì, kèm con số.
- Vì sao nó đáng quan tâm.
- Một câu hỏi nên đào tiếp.
Chỉ kết luận từ dữ liệu có thật. Nếu một mẫu hình có thể chỉ là
ngẫu nhiên, hãy nói rõ.
💡

Luôn yêu cầu Claude nêu rõ giả định. Phần lớn phân tích sai không phải vì tính sai, mà vì định nghĩa khác nhau — "doanh thu" tính trước hay sau hoàn tiền? "Người dùng hoạt động" trong 7 hay 30 ngày? Lộ giả định ra là bắt được lỗi.

04 Viết SQL bằng ngôn ngữ tự nhiên

Nếu dữ liệu của bạn nằm trong cơ sở dữ liệu, Claude là cầu nối giữa câu hỏi và truy vấn.

Cho Claude biết cấu trúc bảng trước

Claude viết SQL tốt tới đâu phụ thuộc nó hiểu lược đồ (schema) tới đâu. Luôn đưa cấu trúc bảng trước:

Prompt — sinh truy vấn SQL
Tôi cần một truy vấn SQL ([loại CSDL: PostgreSQL / MySQL / ...]).

LƯỢC ĐỒ:
"""
[dán câu lệnh tạo bảng, hoặc mô tả: tên bảng, các cột, kiểu, khóa,
quan hệ giữa các bảng]
"""

CÂU HỎI: [diễn đạt bằng lời điều bạn muốn lấy ra]

Yêu cầu:
- Viết truy vấn, kèm chú thích từng phần làm gì.
- Nêu mọi giả định về ý nghĩa dữ liệu.
- Cảnh báo nếu truy vấn có thể chậm trên dữ liệu lớn, và cách tối ưu.
- Nếu câu hỏi của tôi mơ hồ, hỏi lại trước khi viết.

Hiểu & sửa truy vấn có sẵn

Prompt — giải thích SQL
Giải thích truy vấn SQL này từng bước, như cho người mới học SQL:
"""
[dán truy vấn]
"""
- Nó trả về gì?
- Có lỗi tiềm ẩn hoặc trường hợp biên nào không (vd: NULL,
  nhân đôi dòng do JOIN)?
- Viết lại cho dễ đọc hơn mà kết quả không đổi.
⚠️

Luôn kiểm tra truy vấn trên dữ liệu thử trước khi chạy thật. Đặc biệt cẩn trọng với lệnh UPDATE / DELETE — yêu cầu Claude viết kèm câu SELECT tương ứng để bạn xem trước những dòng sẽ bị ảnh hưởng.

05 Quy trình Python / pandas

Cho việc lặp lại hoặc dữ liệu lớn: để Claude viết script tái dùng được, thay vì làm thủ công mỗi lần.

Khi cùng một loại file đến đều đặn — báo cáo bán hàng hàng tháng, xuất dữ liệu hàng tuần — hãy nhờ Claude (qua Claude Code) dựng một script:

Prompt — script xử lý tái dùng
Tôi cần một script Python xử lý lặp lại được.

ĐẦU VÀO: file [định dạng] có cấu trúc: [mô tả các cột].
XỬ LÝ:
1. [bước — vd: làm sạch theo các quy tắc...]
2. [bước — vd: gộp nhóm theo... và tính...]
3. [bước — vd: đối chiếu với file tháng trước]
ĐẦU RA: [mô tả — vd: file Excel có các sheet...]

Yêu cầu:
- Code rõ ràng, có chú thích, người không rành Python vẫn theo được.
- Báo lỗi rõ ràng nếu file đầu vào sai cấu trúc.
- In ra tóm tắt mỗi lần chạy (đã xử lý bao nhiêu dòng, v.v.).
- Hướng dẫn cách chạy script ở cuối.
💡

Một việc thủ công lặp lại hàng tháng, mất vài giờ — biến thành script một lần, sau đó chạy trong vài giây. Đây là khoản đầu tư hoàn vốn rõ nhất ở module Data. Đừng quên nhờ Claude xử lý cả trường hợp file đầu vào bị lỗi.

06 Trích xuất dữ liệu có cấu trúc từ text lộn xộn

Đây là một trong những việc Claude làm tốt nhất: biến văn bản tự do thành bảng dùng được.

Hóa đơn, email, ghi chú phỏng vấn, bài đăng, mô tả sản phẩm — Claude đọc văn bản phi cấu trúc và rút ra trường dữ liệu:

Prompt — trích xuất có cấu trúc
Trích xuất thông tin có cấu trúc từ các văn bản dưới đây.

CẦN LẤY các trường: [liệt kê — vd: tên, công ty, ngày, số tiền,
loại yêu cầu, mức ưu tiên].

QUY TẮC
- Nếu một trường không có trong văn bản, ghi "không có" — KHÔNG đoán.
- [Trường X] phải thuộc một trong: [danh sách giá trị hợp lệ].
- Ngày đưa về dạng YYYY-MM-DD.

ĐỊNH DẠNG ĐẦU RA: bảng (hoặc JSON), mỗi văn bản một dòng.

VĂN BẢN:
"""
[dán các văn bản]
"""
⚠️

Câu "nếu không có thì ghi 'không có', không đoán" rất quan trọng. Không có nó, Claude có thể "điền vào chỗ trống" bằng suy đoán hợp lý nhưng sai. Với lô lớn, hãy kiểm tay một mẫu ngẫu nhiên để ước lượng độ chính xác trước khi tin cả lô.

07 Trực quan hóa & dashboard

Một biểu đồ đúng nói nhiều hơn một bảng số. Claude giúp bạn chọn đúng dạng và dựng nó.

Chọn đúng loại biểu đồ

Prompt — tư vấn trực quan hóa
Tôi muốn trình bày dữ liệu này: [mô tả dữ liệu].
Thông điệp tôi muốn người xem nhận ra: [insight chính].
Người xem là: [vd: ban lãnh đạo / khách hàng / nhóm kỹ thuật].

1. Đề xuất loại biểu đồ phù hợp nhất, giải thích vì sao.
2. Nêu loại biểu đồ NÊN TRÁNH cho dữ liệu này, vì sao.
3. Góp ý về nhãn, chú thích, cách làm nổi điểm chính.

Dựng dashboard tương tác

Claude có thể tạo thẳng một dashboard HTML — biểu đồ tương tác, bộ lọc — xem được trên trình duyệt, không cần cài gì:

Prompt — dashboard HTML
Tạo một dashboard HTML một-file từ dữ liệu đính kèm.

CẦN HIỂN THỊ
- Các chỉ số chính ở trên cùng: [liệt kê].
- Biểu đồ: [mô tả từng biểu đồ và mục đích].
- Bộ lọc: [vd: theo khoảng thời gian, theo nhóm].

YÊU CẦU
- Một file HTML duy nhất, mở trực tiếp bằng trình duyệt.
- Gọn gàng, dễ đọc, hoạt động tốt trên cả điện thoại.
- Nhúng sẵn dữ liệu vào file để không cần kết nối ngoài.
🔗

Cần dashboard đẹp và có cấu trúc hơn? Kỹ thuật xây giao diện HTML nằm ở module Coding — kết hợp hai module để có công cụ nội bộ thật sự dùng được.

08 Báo cáo phân tích tự động

Đỉnh cao của module: đi thẳng từ dữ liệu thô tới một bản tóm tắt mà người ra quyết định đọc được.

Prompt — báo cáo phân tích
Từ dữ liệu đính kèm, soạn một báo cáo phân tích cho [người đọc].

QUY TRÌNH
1. Viết code, chạy thật để tính mọi con số — không ước lượng.
2. Sau đó viết báo cáo gồm:
   - Tóm tắt điều hành: 3–5 ý chính, mỗi ý có số liệu chống lưng.
   - Phát hiện chi tiết, từng cái kèm dữ kiện.
   - Điều dữ liệu KHÔNG nói được, hoặc cảnh báo về độ tin cậy.
   - Đề xuất hành động, gắn với từng phát hiện.

NGUYÊN TẮC
- Mỗi khẳng định phải truy được về một con số đã tính.
- Tách rõ "dữ liệu cho thấy" và "tôi suy luận rằng".
- Viết cho người không chuyên về dữ liệu: ưu tiên rõ ràng.

Khi báo cáo này lặp lại định kỳ, hãy đưa nó thành một quy trình cố định — như mục tự động hóa ở các module trước: cùng một cấu trúc prompt, chỉ thay dữ liệu của kỳ mới.

🎯

Giá trị thật của module Data không phải là "biết phân tích" — mà là rút ngắn quãng đường từ dữ liệu thô tới quyết định. Một báo cáo dù tinh xảo mà không đổi được hành động thì vẫn vô ích. Luôn kết bằng phần "nên làm gì".

★ Bài tập module Data
  1. Chẩn đoán một bộ dữ liệuLấy một file dữ liệu thật, chạy prompt chẩn đoán, đọc báo cáo chất lượng.
  2. Trả lời 3 câu hỏi bằng codeĐặt 3 câu hỏi kinh doanh cho dữ liệu đó, bắt Claude viết code và chạy — đọc kỹ phần giả định.
  3. Trích xuất có cấu trúcLấy một loạt văn bản lộn xộn, biến thành bảng. Kiểm tay một mẫu để đánh giá độ chính xác.
  4. Một script tái dùngChọn một việc xử lý dữ liệu bạn làm lặp lại. Nhờ Claude viết script. Chạy lại tháng sau.
  5. Một báo cáo hoàn chỉnhĐi từ dữ liệu thô tới báo cáo có tóm tắt điều hành và đề xuất hành động.
⚠️

Lỗi thường gặp cần tránh: để Claude "nhẩm" con số quan trọng thay vì chạy code · không hỏi rõ giả định · ghi đè file gốc · tin kết quả trích xuất cả lô mà chưa kiểm mẫu · làm báo cáo đẹp nhưng quên phần "nên làm gì".
